HTML Lists & Tables Guide

HTML Lists

HTML provides three types of lists to organize content effectively: unordered lists, ordered lists, and description lists. Each serves a different purpose in structuring information.

Unordered Lists

Unordered lists (<ul>) are used when the order of items doesn't matter. Each item is marked with a bullet point by default.

Example:

  • First item
  • Second item
  • Third item
<ul> <li>First item</li> <li>Second item</li> <li>Third item</li> </ul>

Ordered Lists

Ordered lists (<ol>) are used when items need to be in a specific sequence. Items are automatically numbered.

Example:

  1. First step
  2. Second step
  3. Third step
<ol> <li>First step</li> <li>Second step</li> <li>Third step</li> </ol>

Description Lists

Description lists (<dl>) are used to create a list of terms and their descriptions.

Example:

HTML
HyperText Markup Language
CSS
Cascading Style Sheets
<dl> <dt>HTML</dt> <dd>HyperText Markup Language</dd> <dt>CSS</dt> <dd>Cascading Style Sheets</dd> </dl>

HTML Tables

Tables are used to organize data into rows and columns. They are created using the <table> element along with related tags.

Basic Table Structure

Key table elements:

  • <table> - Defines a table
  • <tr> - Defines a table row
  • <th> - Defines a table header cell
  • <td> - Defines a table data cell

Table Example

Header 1 Header 2 Header 3
Data 1 Data 2 Data 3
Data 4 Data 5 Data 6
<table> <thead> <tr> <th>Header 1</th> <th>Header 2</th> <th>Header 3</th> </tr> </thead> <tbody> <tr> <td>Data 1</td> <td>Data 2</td> <td>Data 3</td> </tr> </tbody> </table>

Table Attributes

Common table attributes include:

  • colspan - Spans a cell across multiple columns
  • rowspan - Spans a cell across multiple rows
  • border - Specifies table border width
  • cellpadding - Specifies padding within cells
  • cellspacing - Specifies spacing between cells

Best Practices

  • Use tables only for tabular data, not for layout
  • Include proper table headers using <th>
  • Use <thead>, <tbody>, and <tfoot> for better structure
  • Consider responsive design for mobile devices
  • Use CSS for styling rather than HTML attributes
