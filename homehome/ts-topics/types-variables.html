Types & Variables in TypeScript

TypeScript adds static typing to JavaScript, allowing you to specify the type of variables, function parameters, and return values. This helps catch errors during development and provides better tooling support.

Basic Types

TypeScript provides several basic types that you can use to define variables:

Type Description Example
boolean True or false values let isDone: boolean = false;
number Floating point values (integers and decimals) let decimal: number = 6;
string Text values let color: string = "blue";
array Collection of values of the same type let list: number[] = [1, 2, 3];
let list: Array<number> = [1, 2, 3];
tuple Array with fixed number of elements of known types let x: [string, number] = ["hello", 10];
enum Set of named constants enum Color {Red, Green, Blue};
let c: Color = Color.Green;
any Any type (opt-out of type checking) let notSure: any = 4;
void Absence of any type (often used as function return type) function warnUser(): void { console.log("Warning"); }
null and undefined Values that mean "no value" let u: undefined = undefined;
let n: null = null;
never Represents values that never occur function error(message: string): never { throw new Error(message); }
object Non-primitive type let obj: object = { key: "value" };

Variable Declarations

TypeScript supports the same variable declaration keywords as JavaScript, with the addition of type annotations:

var, let, and const

// var: function-scoped, can be redeclared and updated
var x = 10;
var x = 20; // Valid redeclaration

// let: block-scoped, can be updated but not redeclared
let y = 10;
y = 20; // Valid update
// let y = 30; // Error: Cannot redeclare block-scoped variable

// const: block-scoped, cannot be updated or redeclared
const z = 10;
// z = 20; // Error: Cannot assign to 'z' because it is a constant

Best Practice

In TypeScript, it's recommended to:

  • Use const by default
  • Use let when you need to reassign variables
  • Avoid var due to its function scope and hoisting behavior

Type Annotations

Type annotations in TypeScript are a way to explicitly specify the type of a variable, parameter, or return value:

// Basic type annotations
let name: string = "John";
let age: number = 30;
let isActive: boolean = true;

// Array type annotations
let numbers: number[] = [1, 2, 3, 4, 5];
let names: Array<string> = ["Alice", "Bob", "Charlie"];

// Object type annotations
let point: { x: number; y: number } = { x: 10, y: 20 };

Type Inference

TypeScript can often infer the type of a variable based on its initial value, making explicit type annotations optional in many cases:

// Type inference
let name = "John";          // TypeScript infers type as string
let age = 30;               // TypeScript infers type as number
let isActive = true;        // TypeScript infers type as boolean
let numbers = [1, 2, 3];    // TypeScript infers type as number[]

With Type Annotation

let name: string = "John";
let age: number = 30;
let isActive: boolean = true;

With Type Inference

let name = "John";
let age = 30;
let isActive = true;

Union Types

Union types allow a variable to have more than one type:

// Union type
let id: string | number;
id = 101;      // Valid
id = "202";    // Also valid
// id = true;  // Error: Type 'boolean' is not assignable to type 'string | number'

Type Aliases

Type aliases create a new name for a type, making complex types more readable and reusable:

// Type alias
type ID = string | number;
let userId: ID = 123;
let orderId: ID = "A123";

// Object type alias
type Point = {
    x: number;
    y: number;
};

let position: Point = { x: 10, y: 20 };

Literal Types

Literal types allow you to specify exact values that a variable can have:

// String literal type
type Direction = "north" | "south" | "east" | "west";
let direction: Direction = "north";  // Valid
// direction = "northeast";  // Error: Type '"northeast"' is not assignable to type 'Direction'

// Numeric literal type
type DiceRoll = 1 | 2 | 3 | 4 | 5 | 6;
let roll: DiceRoll = 3;  // Valid
// let roll: DiceRoll = 7;  // Error: Type '7' is not assignable to type 'DiceRoll'

Type Assertions

Type assertions are a way to tell the TypeScript compiler that you know better than it about the type of a value:

// Type assertion using angle bracket syntax
let someValue: any = "this is a string";
let strLength: number = (<string>someValue).length;

// Type assertion using 'as' syntax (preferred, especially in JSX)
let someValue2: any = "this is a string";
let strLength2: number = (someValue2 as string).length;

Important Note

Type assertions do not perform any special checking or restructuring of data. They have no runtime impact and are used purely by the compiler.

Non-null Assertion Operator

The non-null assertion operator (!) tells TypeScript that a value cannot be null or undefined:

// Non-null assertion operator
function getLength(s: string | null) {
    // TypeScript would normally error here because s might be null
    return s!.length;  // The ! tells TypeScript that s is not null
}

Differences from JavaScript

Feature JavaScript TypeScript
Type System Dynamic typing (types determined at runtime) Static typing (types checked at compile time)
Type Declarations No explicit type declarations Supports explicit type annotations
Type Errors Only detected at runtime Detected during development
IDE Support Limited intellisense and autocompletion Rich intellisense and autocompletion based on types
