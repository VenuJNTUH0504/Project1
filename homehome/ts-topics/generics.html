TypeScript Generics

Creating reusable components with type safety

What are Generics?

Generics are a way to create reusable components that can work with a variety of types rather than a single one. They allow you to write flexible, reusable functions and classes without sacrificing type safety.

With generics, you can create components that work over a variety of types rather than a single one, which helps to create more reusable code while maintaining type safety.

// A simple generic function
function identity<T>(arg: T): T {
    return arg;
}

// Using the generic function
let output1 = identity<string>("myString");  // type of output will be 'string'
let output2 = identity<number>(100);        // type of output will be 'number'

// Type inference - compiler will figure out the type
let output3 = identity("myString");  // type of output will be 'string'

Generic Interfaces and Classes

You can also use generics with interfaces and classes to create reusable components:

// Generic interface
interface GenericIdentityFn<T> {
    (arg: T): T;
}

// Generic class
class GenericNumber<T> {
    zeroValue: T;
    add: (x: T, y: T) => T;
}

// Using the generic class
let myGenericNumber = new GenericNumber<number>();
myGenericNumber.zeroValue = 0;
myGenericNumber.add = function(x, y) { return x + y; };

Generic Constraints

Sometimes you want to limit the types that a generic type parameter can accept. You can do this using constraints:

// Creating a constraint using an interface
interface Lengthwise {
    length: number;
}

// Using the constraint in a generic function
function loggingIdentity<T extends Lengthwise>(arg: T): T {
    console.log(arg.length);  // Now we know it has a .length property
    return arg;
}

// This works
loggingIdentity({ length: 10, value: 3 });

// This would error because number doesn't have a .length property
// loggingIdentity(3);

Using Type Parameters in Generic Constraints

You can declare a type parameter that is constrained by another type parameter:

function getProperty<T, K extends keyof T>(obj: T, key: K) {
    return obj[key];
}

let x = { a: 1, b: 2, c: 3, d: 4 };

getProperty(x, "a"); // okay
getProperty(x, "m"); // error: Argument of type 'm' isn't assignable to 'a' | 'b' | 'c' | 'd'

Generic Type Aliases

You can also create generic type aliases:

type Container<T> = { value: T };

let numberContainer: Container<number> = { value: 0 };
let stringContainer: Container<string> = { value: "Hello" };

Practical Examples

// Generic function to create a typed array
function createArray<T>(length: number, value: T): T[] {
    return Array(length).fill(value);
}

// Create an array of strings
const stringArray = createArray(3, "hi");  // returns ["hi", "hi", "hi"]

// Create an array of numbers
const numberArray = createArray(3, 42);    // returns [42, 42, 42]

// Generic function to reverse an array
function reverseArray<T>(array: T[]): T[] {
    return array.slice().reverse();
}

// Reverse a string array
const reversedStrings = reverseArray(["a", "b", "c"]);  // returns ["c", "b", "a"]

// Reverse a number array
const reversedNumbers = reverseArray([1, 2, 3]);        // returns [3, 2, 1]
