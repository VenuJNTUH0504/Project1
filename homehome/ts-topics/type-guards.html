Type Guards in TypeScript

Type guards are expressions that perform a runtime check to guarantee the type of a value in a specific scope. They allow TypeScript to narrow down the type of a variable within conditional blocks, making your code more type-safe and expressive.

The Problem: Union Types

When working with union types, TypeScript only allows operations that are valid for all possible types in the union:

function process(value: string | number) {
    // Error: Property 'toUpperCase' does not exist on type 'string | number'.
    // Property 'toUpperCase' does not exist on type 'number'.
    // value.toUpperCase();
    
    // Error: Property 'toFixed' does not exist on type 'string | number'.
    // Property 'toFixed' does not exist on type 'string'.
    // value.toFixed(2);
}

Type guards solve this problem by narrowing the type within conditional blocks.

typeof Type Guards

The typeof operator can be used as a type guard for JavaScript's primitive types:

function process(value: string | number) {
    if (typeof value === "string") {
        // TypeScript knows value is a string here
        console.log(value.toUpperCase());
    } else {
        // TypeScript knows value is a number here
        console.log(value.toFixed(2));
    }
}

// Works with multiple types
function printValue(value: string | number | boolean | object) {
    if (typeof value === "string") {
        console.log("String: " + value);
    } else if (typeof value === "number") {
        console.log("Number: " + value.toFixed(2));
    } else if (typeof value === "boolean") {
        console.log("Boolean: " + (value ? "Yes" : "No"));
    } else {
        console.log("Object: " + JSON.stringify(value));
    }
}

typeof Operator Return Values

The typeof operator returns the following string literals:

  • "string" for strings
  • "number" for numbers
  • "boolean" for booleans
  • "undefined" for undefined
  • "object" for objects (including null!)
  • "function" for functions
  • "symbol" for symbols
  • "bigint" for BigInt values

instanceof Type Guards

The instanceof operator checks if an object is an instance of a class or constructor function:

class Animal {
    name: string;
    constructor(name: string) {
        this.name = name;
    }
}

class Dog extends Animal {
    bark() {
        console.log("Woof!");
    }
}

class Cat extends Animal {
    meow() {
        console.log("Meow!");
    }
}

function makeSound(animal: Animal) {
    if (animal instanceof Dog) {
        // TypeScript knows animal is a Dog here
        animal.bark();
    } else if (animal instanceof Cat) {
        // TypeScript knows animal is a Cat here
        animal.meow();
    } else {
        console.log("Unknown animal");
    }
}

const dog = new Dog("Rex");
const cat = new Cat("Whiskers");

makeSound(dog);  // Outputs: Woof!
makeSound(cat);  // Outputs: Meow!

in Operator Type Guards

The in operator checks if a property exists on an object and can be used as a type guard:

interface Bird {
    fly(): void;
    layEggs(): void;
}

interface Fish {
    swim(): void;
    layEggs(): void;
}

function move(pet: Bird | Fish) {
    if ("fly" in pet) {
        // TypeScript knows pet is a Bird here
        pet.fly();
    } else {
        // TypeScript knows pet is a Fish here
        pet.swim();
    }
}

const bird: Bird = {
    fly() { console.log("Flying..."); },
    layEggs() { console.log("Laying eggs..."); }
};

const fish: Fish = {
    swim() { console.log("Swimming..."); },
    layEggs() { console.log("Laying eggs..."); }
};

move(bird);  // Outputs: Flying...
move(fish);  // Outputs: Swimming...

User-Defined Type Guards

You can create custom type guard functions using type predicates, which have the form parameterName is Type:

// Type guard function for Fish
function isFish(pet: Fish | Bird): pet is Fish {
    return (pet as Fish).swim !== undefined;
}

// Using the type guard
function move(pet: Fish | Bird) {
    if (isFish(pet)) {
        // TypeScript knows pet is a Fish here
        pet.swim();
    } else {
        // TypeScript knows pet is a Bird here
        pet.fly();
    }
}

// More complex example
interface Car {
    make: string;
    model: string;
    year: number;
    startEngine(): void;
}

interface Bicycle {
    make: string;
    model: string;
    year: number;
    pedal(): void;
}

// Type guard for Car
function isCar(vehicle: Car | Bicycle): vehicle is Car {
    return (vehicle as Car).startEngine !== undefined;
}

function useVehicle(vehicle: Car | Bicycle) {
    const info = `${vehicle.make} ${vehicle.model} (${vehicle.year})`;
    
    if (isCar(vehicle)) {
        console.log(`Car: ${info}`);
        vehicle.startEngine();
    } else {
        console.log(`Bicycle: ${info}`);
        vehicle.pedal();
    }
}

Type Guard Reliability

Make sure your type guards are reliable. In the example above, if a Bicycle also had a startEngine method, the type guard would incorrectly identify it as a Car. Consider using discriminated unions for more reliable type narrowing.

Discriminated Unions

Discriminated unions (also called tagged unions) are a pattern where a common property is used to differentiate between union members:

interface Square {
    kind: "square";
    size: number;
}

interface Rectangle {
    kind: "rectangle";
    width: number;
    height: number;
}

interface Circle {
    kind: "circle";
    radius: number;
}

type Shape = Square | Rectangle | Circle;

function calculateArea(shape: Shape): number {
    switch (shape.kind) {
        case "square":
            // TypeScript knows shape is a Square here
            return shape.size * shape.size;
        case "rectangle":
            // TypeScript knows shape is a Rectangle here
            return shape.width * shape.height;
        case "circle":
            // TypeScript knows shape is a Circle here
            return Math.PI * shape.radius ** 2;
    }
}

Exhaustiveness Checking

You can use TypeScript's never type to ensure all cases are handled in a discriminated union:

function calculateArea(shape: Shape): number {
    switch (shape.kind) {
        case "square":
            return shape.size * shape.size;
        case "rectangle":
            return shape.width * shape.height;
        case "circle":
            return Math.PI * shape.radius ** 2;
        default:
            // This will cause a compile-time error if a new shape is added
            // and not handled above
            const exhaustiveCheck: never = shape;
            return exhaustiveCheck;
    }
}

If you add a new shape type but forget to handle it in the switch statement, TypeScript will generate a compile-time error:

// Adding a new shape
interface Triangle {
    kind: "triangle";
    base: number;
    height: number;
}

// Update the Shape type
type Shape = Square | Rectangle | Circle | Triangle;

// Error: Type 'Triangle' is not assignable to type 'never'
// This happens because we didn't handle the "triangle" case

Type Guards with Nullish Values

Type guards are useful for handling potentially null or undefined values:

function printName(name: string | null | undefined) {
    if (name === null) {
        console.log("Name is null");
    } else if (name === undefined) {
        console.log("Name is undefined");
    } else {
        // TypeScript knows name is a string here
        console.log("Name: " + name.toUpperCase());
    }
}

// Simplified null check
function getLength(text: string | null | undefined) {
    // Non-null assertion operator (!) - use with caution
    // return text!.length;  // Dangerous if text can be null/undefined
    
    // Safe approach with type guard
    if (text) {
        // TypeScript knows text is a string here (not null or undefined)
        return text.length;
    }
    return 0;
}

Type Guards with Arrays

Type guards can be used to check array contents:

// Check if an array is non-empty
function firstElement(arr: T[]): T | undefined {
    if (arr.length > 0) {
        // TypeScript knows arr has at least one element
        return arr[0];
    }
    return undefined;
}

// Type guard for array elements
function isStringArray(arr: unknown[]): arr is string[] {
    return arr.every(item => typeof item === "string");
}

function processItems(items: unknown[]) {
    if (isStringArray(items)) {
        // TypeScript knows items is string[] here
        items.forEach(item => console.log(item.toUpperCase()));
    } else {
        console.log("Not a string array");
    }
}

Type Guards with Assertion Functions

Assertion functions are a special kind of function that throw an error if a condition is not met:

// Assertion function
function assertIsString(value: unknown): asserts value is string {
    if (typeof value !== "string") {
        throw new Error("Value is not a string");
    }
}

function processValue(value: unknown) {
    assertIsString(value);
    
    // TypeScript knows value is a string here
    console.log(value.toUpperCase());
}

// More complex assertion
interface User {
    id: number;
    name: string;
}

function assertIsUser(value: unknown): asserts value is User {
    if (!value || typeof value !== "object") {
        throw new Error("Not an object");
    }
    
    const user = value as any;
    if (typeof user.id !== "number" || typeof user.name !== "string") {
        throw new Error("Not a valid User");
    }
}

function processUser(input: unknown) {
    assertIsUser(input);
    
    // TypeScript knows input is a User here
    console.log(`User ${input.name} has ID ${input.id}`);
}

Type Guards with Generic Types

Type guards can be combined with generics for more flexible type checking:

// Generic type guard
function isOfType(value: unknown, property: keyof T): value is T {
    return (
        value !== null &&
        value !== undefined &&
        typeof value === "object" &&
        property in (value as object)
    );
}

interface Product {
    id: number;
    name: string;
    price: number;
}

interface Order {
    id: number;
    items: string[];
    total: number;
}

function processEntity(entity: unknown) {
    if (isOfType(entity, "price")) {
        // TypeScript knows entity is a Product here
        console.log(`Product: ${entity.name}, Price: $${entity.price}`);
    } else if (isOfType(entity, "items")) {
        // TypeScript knows entity is an Order here
        console.log(`Order with ${entity.items.length} items, Total: $${entity.total}`);
    } else {
        console.log("Unknown entity type");
    }
}

Best Practices for Type Guards

Practice Description
Use discriminated unions when possible They provide the most reliable type narrowing and are easier to maintain
Make type guards robust Ensure your type guards correctly identify types in all cases
Use exhaustiveness checking Helps catch missing cases when types are added or changed
Prefer built-in guards for primitives Use typeof for primitive types and instanceof for classes
Create reusable type guards Define type guard functions that can be used across your codebase
