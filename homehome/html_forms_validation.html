HTML Forms & Validation Guide

Form Validation Overview

Form validation ensures that users enter the correct format of data before submitting a form. HTML5 provides built-in validation attributes that make it easier to validate form inputs on the client side.

Required Fields

The 'required' attribute ensures that a field must be filled out before the form can be submitted.

<input type="text" id="username" required>

Email Validation

The email input type automatically validates email addresses:

<input type="email" id="email" placeholder="example@domain.com">

Pattern Validation

The pattern attribute allows you to specify a regular expression that the input's value must match:

<input type="text" pattern="[0-9]{3}-[0-9]{3}-[0-9]{4}">

Length Validation

Control the length of input using minlength and maxlength attributes:

<input type="password" minlength="8" maxlength="20">

Number Range Validation

The min and max attributes control the range of numeric inputs:

<input type="number" min="18" max="100">

Custom Validation Messages

You can customize validation messages using JavaScript:

const input = document.querySelector('#username'); input.addEventListener('invalid', function(event) { if (event.target.validity.valueMissing) { event.target.setCustomValidity('Please enter a username!'); } });

Validation Best Practices

  • Always use both client-side and server-side validation
  • Provide clear error messages
  • Validate in real-time when possible
  • Use appropriate input types
  • Consider accessibility in validation messages
  • Test validation on different browsers

Common Validation Attributes

  • required - Field must be filled out
  • pattern - Specifies a regular expression
  • min/max - Specifies number ranges
  • minlength/maxlength - Specifies text length
  • type - Specifies input type (email, number, etc.)
  • step - Specifies number intervals
