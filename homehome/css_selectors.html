CSS Selectors Guide

Element Selectors

Element selectors target HTML elements directly by their tag name.

p { color: blue; font-size: 16px; } div { padding: 20px; margin: 10px; }
This is a div element

This is a paragraph element

Class Selectors

Class selectors target elements with specific class attributes. They start with a dot (.) followed by the class name.

.highlight { background-color: yellow; } .button { padding: 10px; border-radius: 5px; }
Element with demo-class
Another element with demo-class

ID Selectors

ID selectors target specific elements with unique ID attributes. They start with a hash (#) followed by the ID name.

#header { background-color: black; color: white; } #main-content { padding: 20px; margin: 10px; }
Element with demo-id

Selector Combinations

/* Multiple Selectors */ h1, h2, h3 { font-family: Arial; } /* Descendant Selector */ .container p { margin: 10px; } /* Child Selector */ div > p { padding: 5px; }

Selector Specificity

Selector Type Specificity Value Example
Element 1 p { }
Class 10 .class-name { }
ID 100 #id-name { }
Inline Style 1000 style="..."

Best Practices

  • Use meaningful class and ID names
  • Avoid overusing ID selectors
  • Keep selectors as simple as possible
  • Use classes for reusable styles
  • Follow naming conventions
  • Consider specificity impact

Common Use Cases

  • Styling specific components
  • Creating reusable classes
  • Targeting unique elements
  • Building modular CSS
  • Implementing themes
  • Responsive design

Performance Tips

  • Avoid deep nesting
  • Use efficient selectors
  • Minimize selector specificity
  • Group related selectors
  • Keep selectors short
  • Use appropriate naming
