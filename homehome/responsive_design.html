Responsive Design Guide

What is Responsive Design?

Responsive web design is an approach to web design that makes web pages render well on a variety of devices and window or screen sizes. It uses HTML and CSS to automatically resize, hide, shrink, or enlarge a website to make it look good on all devices.

Viewport Meta Tag

The viewport meta tag is crucial for responsive design. It tells the browser how to control the page's dimensions on different devices:

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Media Queries

Media queries are the backbone of responsive design. They allow you to apply different styles based on device characteristics:

/* Mobile Devices */ @media (max-width: 768px) { .container { width: 100%; padding: 10px; } } /* Tablets */ @media (min-width: 769px) and (max-width: 1024px) { .container { width: 90%; margin: auto; } }

Common Breakpoints

Device Width Usage
Mobile Phones 320px - 480px Small screen devices
Tablets 481px - 768px Medium screen devices
Laptops 769px - 1024px Large screen devices
Desktops 1025px and above Extra large screens

Responsive Images

Make images responsive using CSS:

img { max-width: 100%; height: auto; }

Mobile-First Approach

Design for mobile devices first, then progressively enhance for larger screens:

  • Start with base styles for mobile devices
  • Add complexity as screen size increases
  • Use min-width media queries
  • Focus on essential content first

Responsive Typography

Use relative units for font sizes:

body { font-size: 16px; /* Base font size */ } h1 { font-size: 2rem; /* Relative to root element */ } p { font-size: 1em; /* Relative to parent element */ }

Best Practices

  • Use relative units (%, rem, em) instead of fixed pixels
  • Test on multiple devices and browsers
  • Optimize images for different screen sizes
  • Keep performance in mind
  • Use appropriate touch targets for mobile devices
  • Consider content hierarchy
