BEM Methodology in CSS

BEM (Block, Element, Modifier) is a popular naming convention for classes in HTML and CSS. It helps create reusable components and code sharing in front-end development.

Core Concepts of BEM

1. Block

A block is an independent entity that represents a meaningful part of the interface.

menu
.menu { ... }

2. Element

An element is a part of a block that has no standalone meaning and is semantically tied to its block.

menu
menu__item
menu__item
.menu__item { ... }

3. Modifier

A modifier is a flag on a block or element. It changes appearance, behavior, or state.

menu
menu__item--active
menu__item
.menu__item--active { ... }

4. BEM Naming Convention

BEM uses a strict naming pattern: block__element--modifier

  • block: .menu
  • element: .menu__item
  • modifier: .menu__item--active

BEM vs Traditional CSS

  • Predictable: Class names describe structure and state
  • Reusable: Components can be reused across projects
  • No conflicts: Avoids CSS selector conflicts
  • Scalable: Easy to maintain large codebases
  • Readability: Clear relationships between HTML and CSS
