Go Advanced Package Management

Advanced techniques for managing Go packages and dependencies

Module Replace

Using module replace directives:

// go.mod
module example.com/myproject

go 1.21

require (
    github.com/example/package v1.2.3
)

// Replace a dependency with a local version
replace github.com/example/package => ../local/package

// Replace with a specific version
replace github.com/example/package => github.com/example/package v1.3.0

// Replace with a fork
replace github.com/example/package => github.com/myfork/package v1.2.3

Private Modules

Working with private modules:

// .gitconfig
[url "ssh://git@github.com/"]
    insteadOf = https://github.com/

// .netrc
machine github.com
login git
password YOUR_PERSONAL_ACCESS_TOKEN

// go.mod
module example.com/myproject

go 1.21

require (
    github.com/private/package v1.0.0
)

Vendor Directory

Using the vendor directory:

// Initialize vendor directory
go mod vendor

// Build with vendor directory
go build -mod=vendor

// Update vendor directory
go mod vendor -v

// Check vendor status
go mod verify

Module Proxy

Setting up a module proxy:

// Configure GOPROXY
export GOPROXY=https://proxy.golang.org,direct

// Use multiple proxies
export GOPROXY=https://proxy1.golang.org,https://proxy2.golang.org,direct

// Private proxy
export GOPROXY=https://private-proxy.example.com,direct

// Configure GONOSUMDB
export GONOSUMDB=*.private.example.com

Module Graph

Analyzing module dependencies:

// View module graph
go mod graph

// Visualize module graph
go mod graph | dot -Tpng > graph.png

// Check module requirements
go mod why -m github.com/example/package

// List all dependencies
go list -m all

// Check for updates
go list -m -u all

Best Practices

  • Use semantic versioning for modules
  • Keep go.mod and go.sum in version control
  • Use specific versions instead of latest
  • Regularly update dependencies
  • Use module replace for local development
  • Configure private module access properly
  • Use vendor directory for reproducible builds
  • Set up appropriate module proxies
  • Document module requirements and constraints
  • Regularly audit dependencies for security
