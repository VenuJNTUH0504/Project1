Understanding functions in Go

Basic Functions

Functions in Go are declared using the func keyword:

// Simple function
func greet() {
    fmt.Println("Hello, World!")
}

// Function with parameters
func add(a int, b int) int {
    return a + b
}

// Function with named return value
func divide(a, b float64) (result float64) {
    result = a / b
    return
}

// Function with multiple return values
func swap(a, b string) (string, string) {
    return b, a
}

Variadic Functions

Functions that can accept a variable number of arguments:

// Variadic function
func sum(numbers ...int) int {
    total := 0
    for _, num := range numbers {
        total += num
    }
    return total
}

// Using variadic function
func main() {
    fmt.Println(sum(1, 2, 3))        // 6
    fmt.Println(sum(1, 2, 3, 4, 5))  // 15
}

Function Types

Functions are first-class citizens in Go:

// Function type
type Operation func(int, int) int

// Using function type
func calculate(a, b int, op Operation) int {
    return op(a, b)
}

// Example usage
func main() {
    add := func(a, b int) int { return a + b }
    multiply := func(a, b int) int { return a * b }
    
    fmt.Println(calculate(5, 3, add))      // 8
    fmt.Println(calculate(5, 3, multiply)) // 15
}

Closures

Functions can form closures with their surrounding scope:

// Counter using closure
func counter() func() int {
    count := 0
    return func() int {
        count++
        return count
    }
}

// Using closure
func main() {
    c := counter()
    fmt.Println(c()) // 1
    fmt.Println(c()) // 2
    fmt.Println(c()) // 3
}

Deferred Functions

Functions can be deferred to execute after the surrounding function returns:

func processFile(filename string) error {
    file, err := os.Open(filename)
    if err != nil {
        return err
    }
    defer file.Close()
    
    // Process file
    return nil
}

// Multiple defer statements
func main() {
    defer fmt.Println("Third")
    defer fmt.Println("Second")
    defer fmt.Println("First")
    // Output: First Second Third
}

Best Practices

  • Keep functions small and focused
  • Use meaningful function names
  • Return errors as the last return value
  • Use named return values when they improve clarity
  • Document exported functions
  • Use defer for cleanup operations
  • Consider using interfaces for function parameters
  • Handle errors appropriately
