Understanding control flow in Go

If-Else Statements

Go's if-else statements are similar to other languages but with some unique features:

// Basic if statement
if x > 0 {
    fmt.Println("x is positive")
}

// If-else statement
if x > 0 {
    fmt.Println("x is positive")
} else {
    fmt.Println("x is not positive")
}

// If-else if-else statement
if x > 0 {
    fmt.Println("x is positive")
} else if x < 0 {
    fmt.Println("x is negative")
} else {
    fmt.Println("x is zero")
}

// If with initialization
if x := 10; x > 0 {
    fmt.Println("x is positive")
}

Switch Statements

Go's switch statements are more flexible than in many other languages:

// Basic switch
switch day {
case "Monday":
    fmt.Println("Start of the week")
case "Friday":
    fmt.Println("End of the week")
default:
    fmt.Println("Midweek")
}

// Switch with initialization
switch x := 10; {
case x > 0:
    fmt.Println("Positive")
case x < 0:
    fmt.Println("Negative")
default:
    fmt.Println("Zero")
}

// Type switch
var i interface{} = "hello"
switch v := i.(type) {
case int:
    fmt.Printf("Integer: %v\n", v)
case string:
    fmt.Printf("String: %v\n", v)
default:
    fmt.Printf("Unknown type: %T\n", v)
}

For Loops

Go has only one loop construct, the for loop, which is very versatile:

// Basic for loop
for i := 0; i < 10; i++ {
    fmt.Println(i)
}

// While-like loop
i := 0
for i < 10 {
    fmt.Println(i)
    i++
}

// Infinite loop
for {
    // Do something
    if condition {
        break
    }
}

// Range loop
numbers := []int{1, 2, 3, 4, 5}
for index, value := range numbers {
    fmt.Printf("Index: %d, Value: %d\n", index, value)
}

Break and Continue

Control loop execution with break and continue:

// Break example
for i := 0; i < 10; i++ {
    if i == 5 {
        break
    }
    fmt.Println(i)
}

// Continue example
for i := 0; i < 10; i++ {
    if i%2 == 0 {
        continue
    }
    fmt.Println(i)
}

// Break with label
outer:
for i := 0; i < 3; i++ {
    for j := 0; j < 3; j++ {
        if i == 1 && j == 1 {
            break outer
        }
        fmt.Printf("i: %d, j: %d\n", i, j)
    }
}

Defer Statement

The defer statement postpones the execution of a function until the surrounding function returns:

func main() {
    defer fmt.Println("This will be printed last")
    fmt.Println("This will be printed first")
}

// Multiple defers are executed in LIFO order
func main() {
    defer fmt.Println("Third")
    defer fmt.Println("Second")
    defer fmt.Println("First")
}

// Common use case: closing resources
func readFile() error {
    file, err := os.Open("file.txt")
    if err != nil {
        return err
    }
    defer file.Close()
    // Use file
    return nil
}

Best Practices

  • Use early returns to reduce nesting
  • Keep if conditions simple and readable
  • Use switch statements for multiple conditions
  • Prefer range loops when iterating over collections
  • Use defer for cleanup operations
  • Be careful with infinite loops
  • Use meaningful loop variable names
  • Consider using break labels for complex nested loops
