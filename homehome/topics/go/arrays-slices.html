Go Arrays & Slices

Understanding arrays and slices in Go

Arrays

Arrays in Go are fixed-size sequences of elements of the same type:

// Array declaration
var numbers [5]int
numbers[0] = 1
numbers[1] = 2

// Array initialization
primes := [6]int{2, 3, 5, 7, 11, 13}

// Array with ellipsis
days := [...]string{"Monday", "Tuesday", "Wednesday"}

// Multi-dimensional array
var matrix [3][3]int
matrix[0][0] = 1
matrix[1][1] = 1
matrix[2][2] = 1

Slices

Slices are dynamic views into arrays or other slices:

// Slice declaration
var s []int

// Slice from array
primes := [6]int{2, 3, 5, 7, 11, 13}
s := primes[1:4] // [3, 5, 7]

// Slice literal
s := []int{1, 2, 3, 4, 5}

// Slice with make
s := make([]int, 5)     // length = 5, capacity = 5
s := make([]int, 0, 5)  // length = 0, capacity = 5

Slice Operations

Common operations on slices:

// Append elements
s := []int{1, 2, 3}
s = append(s, 4, 5) // [1, 2, 3, 4, 5]

// Copy slices
src := []int{1, 2, 3}
dst := make([]int, len(src))
copy(dst, src)

// Slice expressions
s := []int{1, 2, 3, 4, 5}
s[1:3]  // [2, 3]
s[:3]   // [1, 2, 3]
s[3:]   // [4, 5]
s[:]    // [1, 2, 3, 4, 5]

Slice Capacity

Understanding slice capacity and length:

// Length and capacity
s := make([]int, 3, 5)
fmt.Println(len(s)) // 3
fmt.Println(cap(s)) // 5

// Growing slices
s := []int{1, 2, 3}
fmt.Println(len(s), cap(s)) // 3, 3
s = append(s, 4)
fmt.Println(len(s), cap(s)) // 4, 6

// Slicing and capacity
s := []int{1, 2, 3, 4, 5}
s1 := s[1:3]
fmt.Println(len(s1), cap(s1)) // 2, 4

Slice Iteration

Different ways to iterate over slices:

// Basic for loop
s := []int{1, 2, 3}
for i := 0; i < len(s); i++ {
    fmt.Println(s[i])
}

// Range loop
s := []int{1, 2, 3}
for index, value := range s {
    fmt.Printf("Index: %d, Value: %d\n", index, value)
}

// Range loop without index
s := []int{1, 2, 3}
for _, value := range s {
    fmt.Println(value)
}

Best Practices

  • Prefer slices over arrays for most use cases
  • Use make when you know the capacity in advance
  • Be careful with slice capacity when appending
  • Use copy when you need to create independent slices
  • Consider the performance implications of slice operations
  • Use range for iteration when possible
  • Be aware of slice aliasing
  • Document slice capacity requirements
