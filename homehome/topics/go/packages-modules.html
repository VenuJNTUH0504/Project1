Go Packages & Modules

Understanding packages and modules in Go

Packages

Packages are the basic unit of code organization in Go:

// Package declaration
package main

// Importing packages
import (
    "fmt"
    "math"
    "github.com/user/repo"
)

// Using imported packages
func main() {
    fmt.Println(math.Sqrt(16))
    repo.DoSomething()
}

Modules

Modules are collections of related Go packages:

// Initialize a module
go mod init github.com/user/repo

// Module file (go.mod)
module github.com/user/repo

go 1.21

require (
    github.com/some/dependency v1.2.3
    github.com/another/dependency v2.0.0
)

// Adding dependencies
go get github.com/some/dependency@v1.2.3

Package Organization

Best practices for organizing packages:

// Project structure
project/
├── cmd/
│   └── app/
│       └── main.go
├── internal/
│   ├── config/
│   └── server/
├── pkg/
│   ├── api/
│   └── utils/
└── go.mod

// Package naming
package config
package server
package api
package utils

Dependency Management

Managing dependencies with Go modules:

// Updating dependencies
go get -u github.com/some/dependency
go mod tidy

// Vendor dependencies
go mod vendor

// Checking dependencies
go list -m all
go mod graph

// Updating module
go mod edit -go 1.21

Package Documentation

Documenting packages and their contents:

// Package documentation
// Package config provides configuration management.
// It supports loading configuration from various sources.
package config

// Function documentation
// LoadConfig loads configuration from the specified path.
// It returns an error if the configuration cannot be loaded.
func LoadConfig(path string) (*Config, error) {
    // ...
}

// Type documentation
// Config represents application configuration.
type Config struct {
    // Server configuration
    Server struct {
        Host string
        Port int
    }
}

Best Practices

  • Use meaningful package names
  • Keep packages focused and cohesive
  • Use internal packages for private code
  • Document exported types and functions
  • Follow semantic versioning
  • Use go.mod for dependency management
  • Keep dependencies up to date
  • Use vendor for reproducible builds
