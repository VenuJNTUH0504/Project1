Advanced OOP Concepts in C++

Mastering advanced object-oriented programming techniques in C++

Multiple Inheritance

Multiple inheritance allows a class to inherit from more than one base class. This powerful feature enables complex class hierarchies but requires careful management to avoid ambiguity.

class Animal {
public:
    virtual void eat() = 0;
};

class Mammal {
public:
    virtual void breathe() = 0;
};

class Bat : public Animal, public Mammal {
public:
    void eat() override { /* implementation */ }
    void breathe() override { /* implementation */ }
};

Warning: Multiple inheritance can lead to the "diamond problem" when two base classes inherit from a common ancestor.

Virtual Inheritance

Virtual inheritance solves the diamond problem by ensuring that only one instance of the common base class exists in the inheritance hierarchy.

class Animal {
public:
    virtual void eat() = 0;
};

class Mammal : virtual public Animal {
    // Mammal implementation
};

class Bird : virtual public Animal {
    // Bird implementation
};

class Bat : public Mammal, public Bird {
    // Bat implementation
};

Note: Virtual inheritance adds some overhead and should be used only when necessary to resolve ambiguity.

Abstract Classes and Interfaces

Abstract classes provide a way to define interfaces and common functionality that derived classes must implement.

class Shape {
public:
    virtual double area() const = 0;  // Pure virtual function
    virtual double perimeter() const = 0;
    virtual ~Shape() = default;  // Virtual destructor
};

class Circle : public Shape {
private:
    double radius;
public:
    Circle(double r) : radius(r) {}
    double area() const override { return 3.14159 * radius * radius; }
    double perimeter() const override { return 2 * 3.14159 * radius; }
};

Polymorphism and Virtual Functions

Polymorphism allows objects of different classes to be treated uniformly through a common interface.

class Animal {
public:
    virtual void makeSound() { cout << "Some sound" << endl; }
    virtual ~Animal() = default;
};

class Dog : public Animal {
public:
    void makeSound() override { cout << "Woof!" << endl; }
};

class Cat : public Animal {
public:
    void makeSound() override { cout << "Meow!" << endl; }
};

void makeAnimalSound(Animal& animal) {
    animal.makeSound();  // Calls the appropriate override
}

Friend Classes and Functions

Friend declarations allow specific classes or functions to access private and protected members of a class.

class Matrix {
private:
    int data[4][4];
    friend class MatrixOperations;
    friend void printMatrix(const Matrix& m);
};

class MatrixOperations {
public:
    static Matrix multiply(const Matrix& a, const Matrix& b) {
        Matrix result;
        // Can access private members of Matrix
        return result;
    }
};

void printMatrix(const Matrix& m) {
    // Can access private members of Matrix
}

Best Practice: Use friend declarations sparingly as they break encapsulation. Consider alternative designs first.

Operator Overloading

Operator overloading allows you to define custom behavior for operators when used with your classes.

class Complex {
private:
    double real, imag;
public:
    Complex operator+(const Complex& other) const {
        return Complex(real + other.real, imag + other.imag);
    }
    
    Complex& operator+=(const Complex& other) {
        real += other.real;
        imag += other.imag;
        return *this;
    }
    
    friend std::ostream& operator<<(std::ostream& os, const Complex& c);
};

std::ostream& operator<<(std::ostream& os, const Complex& c) {
    os << c.real << " + " << c.imag << "i";
    return os;
}
