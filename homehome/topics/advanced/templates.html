Templates in C++

Understanding and implementing generic programming with templates

Function Templates

Function templates allow you to write generic functions that work with different data types.

template 
T max(T a, T b) {
    return (a > b) ? a : b;
}

// Usage
int i = max(10, 20);
double d = max(3.14, 2.71);
string s = max("apple", "banana");

Note: The compiler generates specific versions of the template function for each type used.

Class Templates

Class templates allow you to create generic classes that can work with different data types.

template 
class Stack {
private:
    std::vector elements;
public:
    void push(const T& element) {
        elements.push_back(element);
    }
    
    T pop() {
        if (elements.empty()) {
            throw std::out_of_range("Stack is empty");
        }
        T element = elements.back();
        elements.pop_back();
        return element;
    }
};

// Usage
Stack intStack;
Stack stringStack;

Template Specialization

Template specialization allows you to provide specific implementations for particular types.

template 
class Vector {
    // Generic implementation
};

template <>
class Vector {
    // Specialized implementation for bool
    // Uses bit-level storage
};

Best Practice: Use template specialization when you need special handling for specific types.

Variadic Templates

Variadic templates allow functions and classes to accept a variable number of template arguments.

template 
void print(Args... args) {
    (std::cout << ... << args) << std::endl;
}

// Usage
print(1, "Hello", 3.14);  // Prints: 1Hello3.14

Warning: Variadic templates can make code harder to read and debug. Use them judiciously.

Template Metaprogramming

Template metaprogramming allows you to perform computations at compile-time using templates.

template 
struct Factorial {
    static const int value = N * Factorial::value;
};

template <>
struct Factorial<0> {
    static const int value = 1;
};

// Usage
const int fact5 = Factorial<5>::value;  // 120 at compile-time

Note: Template metaprogramming can significantly increase compile times and code complexity.

Concepts (C++20)

Concepts provide a way to specify requirements on template parameters.

template 
concept Addable = requires(T a, T b) {
    { a + b } -> std::convertible_to;
};

template 
T sum(T a, T b) {
    return a + b;
}

Modern C++: Concepts make template code more readable and provide better error messages.
