File I/O in C++

Understanding file operations and stream handling in C++

Introduction to File I/O

File I/O in C++ is handled through streams, which provide a convenient way to read from and write to files. The fstream library provides the necessary classes for file operations.

Note: Always check if file operations succeed and handle potential errors appropriately.

Basic File Operations

Here are the basic operations for working with files in C++:

#include 
#include 

// Writing to a file
ofstream outFile("example.txt");
if (outFile.is_open()) {
    outFile << "Hello, World!" << endl;
    outFile.close();
}

// Reading from a file
ifstream inFile("example.txt");
string line;
if (inFile.is_open()) {
    while (getline(inFile, line)) {
        cout << line << endl;
    }
    inFile.close();
}

Warning: Always close files after use to free system resources and ensure all data is written.

File Modes

Files can be opened in different modes to control how they are accessed:

// Open for writing (truncate if exists)
ofstream file("data.txt", ios::out);

// Open for reading
ifstream file("data.txt", ios::in);

// Open for appending
ofstream file("data.txt", ios::app);

// Open for both reading and writing
fstream file("data.txt", ios::in | ios::out);

Best Practice: Use appropriate file modes based on your needs. For example, use ios::app when you want to add data to an existing file.

Binary File Operations

Binary files store data in its raw form, which is useful for storing complex data structures:

struct Person {
    char name[50];
    int age;
    double salary;
};

// Writing binary data
Person p = {"John Doe", 30, 50000.0};
ofstream outFile("person.dat", ios::binary);
outFile.write(reinterpret_cast(&p), sizeof(Person));
outFile.close();

// Reading binary data
Person p2;
ifstream inFile("person.dat", ios::binary);
inFile.read(reinterpret_cast(&p2), sizeof(Person));
inFile.close();

Warning: Be careful with binary file operations as they are platform-dependent and may not be portable.

Error Handling

Proper error handling is crucial when working with files:

ifstream file("data.txt");
if (!file) {
    cerr << "Error: Could not open file" << endl;
    return 1;
}

// Check file state
if (file.fail()) {
    cerr << "Error: File operation failed" << endl;
}

// Check for end of file
if (file.eof()) {
    cout << "Reached end of file" << endl;
}

Best Practice: Always check file operations for errors and provide meaningful error messages.
