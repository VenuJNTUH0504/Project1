Exception Handling in C++

Understanding error handling and exception management in C++

Introduction to Exceptions

Exception handling in C++ provides a way to handle runtime errors in a structured and controlled manner. It allows you to separate error handling code from the normal flow of your program.

Note: Exceptions should be used for exceptional circumstances, not for normal program flow control.

Basic Exception Handling

The basic exception handling mechanism in C++ consists of three keywords: try, catch, and throw.

try {
    // Code that might throw an exception
    if (error_condition) {
        throw "Error message";
    }
} catch (const char* msg) {
    // Handle the exception
    cerr << "Error: " << msg << endl;
}

Warning: Always catch exceptions by reference to avoid slicing and to maintain polymorphic behavior.

Custom Exception Classes

You can create custom exception classes by inheriting from the standard exception class:

#include 

class MyException : public std::exception {
private:
    string message;
    
public:
    MyException(const string& msg) : message(msg) {}
    
    const char* what() const noexcept override {
        return message.c_str();
    }
};

// Usage
try {
    throw MyException("Custom error message");
} catch (const MyException& e) {
    cerr << "Error: " << e.what() << endl;
}

Best Practice: Create specific exception classes for different types of errors to provide better error handling and debugging information.

Multiple Catch Blocks

You can have multiple catch blocks to handle different types of exceptions:

try {
    // Code that might throw different exceptions
    if (condition1) {
        throw runtime_error("Runtime error");
    }
    if (condition2) {
        throw invalid_argument("Invalid argument");
    }
} catch (const runtime_error& e) {
    cerr << "Runtime error: " << e.what() << endl;
} catch (const invalid_argument& e) {
    cerr << "Invalid argument: " << e.what() << endl;
} catch (...) {
    cerr << "Unknown exception occurred" << endl;
}

Note: The catch-all block (catch (...)) should be the last catch block and used sparingly.

Exception Safety

Exception safety refers to how well a program maintains its invariants when exceptions occur. There are three levels of exception safety:

  • Basic Guarantee: No resources are leaked, and objects remain in a valid state.
  • Strong Guarantee: If an operation fails, the program state remains unchanged.
  • No-throw Guarantee: The operation will never throw an exception.

Warning: Always aim for at least basic exception safety in your code. Use RAII (Resource Acquisition Is Initialization) to ensure proper resource management.
