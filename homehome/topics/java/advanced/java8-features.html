Java 8 Features

Exploring the major features introduced in Java 8

Introduction to Java 8

Java 8 introduced several significant features that changed how we write Java code.

Key features introduced in Java 8:

  • Lambda Expressions
  • Stream API
  • Functional Interfaces
  • Method References
  • Default Methods
  • Optional Class
  • Date and Time API

Lambda Expressions

Lambda expressions provide a clear and concise way to represent a method interface using an expression.

Example: Lambda Expressions

// Before Java 8
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello World");
    }
};

// With Lambda Expressions
Runnable r = () -> System.out.println("Hello World");

// More examples
Comparator comparator = (s1, s2) -> s1.compareTo(s2);
Consumer consumer = s -> System.out.println(s);
Predicate isEven = n -> n % 2 == 0;

Stream API

The Stream API provides a functional approach to processing collections of objects.

Example: Stream Operations

List names = Arrays.asList("John", "Alice", "Bob", "Charlie");

// Filter and collect
List filteredNames = names.stream()
    .filter(name -> name.startsWith("A"))
    .collect(Collectors.toList());

// Map and collect
List nameLengths = names.stream()
    .map(String::length)
    .collect(Collectors.toList());

// Reduce
int totalLength = names.stream()
    .mapToInt(String::length)
    .sum();

// Sort
List sortedNames = names.stream()
    .sorted()
    .collect(Collectors.toList());

// Group by
Map> namesByLength = names.stream()
    .collect(Collectors.groupingBy(String::length));

Functional Interfaces

Functional interfaces are interfaces with a single abstract method.

Example: Functional Interfaces

// Built-in functional interfaces
Function stringToInt = Integer::parseInt;
Predicate isLong = s -> s.length() > 10;
Consumer printer = System.out::println;
Supplier stringSupplier = () -> "Hello";
UnaryOperator toUpper = String::toUpperCase;

// Custom functional interface
@FunctionalInterface
interface StringProcessor {
    String process(String input);
    
    // Can have default methods
    default String processTwice(String input) {
        return process(process(input));
    }
}

// Usage
StringProcessor upperCase = String::toUpperCase;
StringProcessor addPrefix = s -> "Prefix_" + s;

Method References

Method references provide a way to refer to methods without executing them.

Example: Method References

// Static method reference
Function parseInt = Integer::parseInt;

// Instance method reference
String str = "Hello";
Supplier length = str::length;

// Constructor reference
Supplier> listSupplier = ArrayList::new;

// Arbitrary object method reference
Function toUpperCase = String::toUpperCase;

// Usage in streams
List names = Arrays.asList("John", "Alice", "Bob");
names.forEach(System.out::println);
List upperNames = names.stream()
    .map(String::toUpperCase)
    .collect(Collectors.toList());

Default Methods

Default methods allow adding new methods to interfaces without breaking existing implementations.

Example: Default Methods

interface Vehicle {
    void start();
    
    default void stop() {
        System.out.println("Vehicle stopped");
    }
}

interface FourWheeler {
    default void print() {
        System.out.println("I am a four wheeler");
    }
}

class Car implements Vehicle, FourWheeler {
    @Override
    public void start() {
        System.out.println("Car started");
    }
    
    // Resolving multiple inheritance of default methods
    @Override
    public void print() {
        Vehicle.super.print();
        FourWheeler.super.print();
        System.out.println("I am a car");
    }
}

Optional Class

Optional is a container object used to represent a value that may or may not be present.

Example: Optional Usage

public class OptionalExample {
    public static void main(String[] args) {
        // Creating Optional
        Optional empty = Optional.empty();
        Optional nonEmpty = Optional.of("Hello");
        Optional nullable = Optional.ofNullable(null);
        
        // Checking presence
        if (nonEmpty.isPresent()) {
            System.out.println(nonEmpty.get());
        }
        
        // Using ifPresent
        nonEmpty.ifPresent(System.out::println);
        
        // Using orElse
        String value = nullable.orElse("Default");
        
        // Using orElseGet
        String value2 = nullable.orElseGet(() -> "Default from supplier");
        
        // Using orElseThrow
        try {
            String value3 = nullable.orElseThrow(() -> 
                new RuntimeException("Value not present"));
        } catch (RuntimeException e) {
            System.out.println(e.getMessage());
        }
        
        // Using map and flatMap
        Optional length = nonEmpty.map(String::length);
        Optional upper = nonEmpty.map(String::toUpperCase);
    }
}

Date and Time API

The new Date and Time API provides a comprehensive model for date and time handling.

Example: Date and Time Operations

import java.time.*;

public class DateTimeExample {
    public static void main(String[] args) {
        // Current date and time
        LocalDate today = LocalDate.now();
        LocalTime now = LocalTime.now();
        LocalDateTime dateTime = LocalDateTime.now();
        
        // Creating specific dates
        LocalDate birthday = LocalDate.of(1990, Month.JANUARY, 1);
        LocalTime meetingTime = LocalTime.of(14, 30);
        
        // Duration and Period
        Duration duration = Duration.between(now, now.plusHours(2));
        Period period = Period.between(today, today.plusDays(7));
        
        // Formatting
        DateTimeFormatter formatter = DateTimeFormatter.ofPattern("dd/MM/yyyy HH:mm");
        String formatted = dateTime.format(formatter);
        
        // Parsing
        LocalDateTime parsed = LocalDateTime.parse("01/01/2020 12:00", formatter);
        
        // Time zones
        ZonedDateTime zonedDateTime = ZonedDateTime.now(ZoneId.of("Europe/Paris"));
    }
}

Java 8 Best Practices

Best practices for using Java 8 features:

  • Use lambda expressions for cleaner, more readable code
  • Prefer method references over lambda expressions when possible
  • Use Stream API for collection processing
  • Use Optional to handle null values
  • Use the new Date and Time API instead of old Date/Calendar
  • Keep lambda expressions short and focused
  • Use parallel streams carefully and only when beneficial
