Advanced Object-Oriented Programming Concepts in Java

Exploring advanced OOP concepts and their implementation in Java

Inheritance

Inheritance is a fundamental OOP concept that allows a class to inherit properties and methods from another class.

Example: Basic Inheritance

class Animal {
    void eat() {
        System.out.println("Animal is eating");
    }
}

class Dog extends Animal {
    void bark() {
        System.out.println("Dog is barking");
    }
}

Key points about inheritance:

  • Java supports single inheritance (a class can extend only one class)
  • All classes implicitly extend Object class
  • Use 'extends' keyword for inheritance

Polymorphism

Polymorphism allows objects to take multiple forms and enables one interface to be used for a general class of actions.

Example: Method Overriding

class Shape {
    void draw() {
        System.out.println("Drawing a shape");
    }
}

class Circle extends Shape {
    @Override
    void draw() {
        System.out.println("Drawing a circle");
    }
}

Types of polymorphism in Java:

  • Compile-time polymorphism (Method overloading)
  • Runtime polymorphism (Method overriding)

Abstraction

Abstraction is the process of hiding implementation details and showing only essential features.

Example: Abstract Class

abstract class Vehicle {
    abstract void start();
    
    void stop() {
        System.out.println("Vehicle stopped");
    }
}

class Car extends Vehicle {
    @Override
    void start() {
        System.out.println("Car started");
    }
}

Abstraction can be achieved through:

  • Abstract classes
  • Interfaces

Encapsulation

Encapsulation is the mechanism of wrapping data and methods together as a single unit.

Example: Encapsulation

class Student {
    private String name;
    private int age;
    
    public String getName() {
        return name;
    }
    
    public void setName(String name) {
        this.name = name;
    }
    
    public int getAge() {
        return age;
    }
    
    public void setAge(int age) {
        if(age > 0) {
            this.age = age;
        }
    }
}

Benefits of encapsulation:

  • Data hiding
  • Increased flexibility
  • Reusability
  • Testing is easier

Interfaces

Interfaces define a contract that classes must implement, specifying what methods a class must have.

Example: Interface Implementation

interface Drawable {
    void draw();
    double getArea();
}

class Rectangle implements Drawable {
    private double width;
    private double height;
    
    @Override
    public void draw() {
        System.out.println("Drawing rectangle");
    }
    
    @Override
    public double getArea() {
        return width * height;
    }
}

Interface features:

  • All methods are public and abstract by default
  • Can have default and static methods (Java 8+)
  • Can have private methods (Java 9+)
  • Supports multiple inheritance

Composition

Composition is a design technique to implement has-a relationship in classes.

Example: Composition

class Engine {
    void start() {
        System.out.println("Engine started");
    }
}

class Car {
    private Engine engine;
    
    Car() {
        engine = new Engine();
    }
    
    void start() {
        engine.start();
    }
}

Composition vs Inheritance:

  • Composition provides better flexibility
  • Easier to change behavior at runtime
  • Better encapsulation
