Spring Framework

Spring is a comprehensive framework for building enterprise Java applications. It provides infrastructure support for developing Java applications, handling tasks like dependency injection, transaction management, web applications, data access, messaging, and more.

Key Features of Spring

Dependency Injection

Spring's core feature is dependency injection (DI), which allows objects to define their dependencies without creating them. The Spring container injects those dependencies when it creates the bean.

Aspect-Oriented Programming

Spring supports Aspect-Oriented Programming (AOP), which enables separation of cross-cutting concerns like logging, security, and transaction management from business logic.

Spring Boot

Spring Boot simplifies the setup and development of Spring applications by providing auto-configuration, standalone applications, and production-ready features.

Enterprise Integration

Spring provides extensive support for enterprise integration, including data access, transaction management, messaging, and integration with various enterprise technologies.

Spring Code Example

Here's a simple example of a Spring Boot REST controller:

package com.example.demo;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;
import org.springframework.web.bind.annotation.*;

@SpringBootApplication
public class DemoApplication {

    public static void main(String[] args) {
        SpringApplication.run(DemoApplication.class, args);
    }
}
package com.example.demo.controller;

import com.example.demo.model.Product;
import com.example.demo.service.ProductService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/products")
public class ProductController {

    private final ProductService productService;

    @Autowired
    public ProductController(ProductService productService) {
        this.productService = productService;
    }

    @GetMapping
    public List<Product> getAllProducts() {
        return productService.findAll();
    }

    @GetMapping("/{id}")
    public Product getProductById(@PathVariable Long id) {
        return productService.findById(id)
                .orElseThrow(() -> new RuntimeException("Product not found"));
    }

    @PostMapping
    public Product createProduct(@RequestBody Product product) {
        return productService.save(product);
    }

    @PutMapping("/{id}")
    public Product updateProduct(@PathVariable Long id, @RequestBody Product product) {
        product.setId(id);
        return productService.save(product);
    }

    @DeleteMapping("/{id}")
    public void deleteProduct(@PathVariable Long id) {
        productService.deleteById(id);
    }
}

Spring Boot vs. Spring Framework

Spring Boot is built on top of the Spring Framework and provides a simpler way to create stand-alone, production-grade Spring-based applications:

Spring Framework

The core framework that provides fundamental support for dependency injection, transaction management, web applications, data access, messaging, and more.

Spring Boot

An extension of the Spring Framework that simplifies the process of creating Spring applications with auto-configuration, embedded servers, and production-ready features.

Getting Started with Spring Boot

To create a new Spring Boot application, you can use Spring Initializer:

# Visit Spring Initializer website
https://start.spring.io/

# Or use Spring CLI
spring init --dependencies=web,data-jpa,h2 my-project

# Or use Maven
mvn spring-boot:run

# Or use Gradle
./gradlew bootRun

After creating your Spring Boot project, you can import it into your IDE and start developing your application. Spring Boot applications typically have a main class with the @SpringBootApplication annotation that serves as the entry point.
