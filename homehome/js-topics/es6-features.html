ES6+ Features

ECMAScript 6 (ES6), also known as ECMAScript 2015, introduced significant enhancements to JavaScript. Subsequent versions (ES7, ES8, ES9, etc.) have continued to add powerful features. This guide covers the most important ES6+ features that have transformed modern JavaScript development.

Key ES6 Features

1. Let and Const Declarations

Block-scoped variables that provide better control over variable scope compared to var.

Before ES6 (using var)

var x = 10;
if (true) {
  var x = 20; // Same variable!
  console.log(x); // 20
}
console.log(x); // 20 - the value changed

ES6 (using let)

let x = 10;
if (true) {
  let x = 20; // Different variable due to block scope
  console.log(x); // 20
}
console.log(x); // 10 - original value preserved

Const for Constants

const creates read-only references, but doesn't make objects immutable:

const PI = 3.14159;
PI = 3.14; // Error: Assignment to constant variable

const user = { name: "John" };
user.name = "Jane"; // Works! Only the reference is constant
user = {}; // Error: Assignment to constant variable

2. Arrow Functions

A more concise syntax for writing functions with lexical this binding.

Before ES6

function add(a, b) {
  return a + b;
}

// With this binding issues
var obj = {
  value: 42,
  getValue: function() {
    var self = this; // Store this
    setTimeout(function() {
      console.log(self.value);
    }, 1000);
  }
};

ES6 Arrow Functions

const add = (a, b) => a + b;

// Lexical this binding
const obj = {
  value: 42,
  getValue: function() {
    setTimeout(() => {
      console.log(this.value); // this refers to obj
    }, 1000);
  }
};

3. Template Literals

Enhanced string literals that allow embedded expressions and multi-line strings.

Before ES6

var name = "John";
var greeting = "Hello " + name + "!";

var multiline = "This is line 1.\n" +
                "This is line 2.\n" +
                "This is line 3.";

ES6 Template Literals

const name = "John";
const greeting = `Hello ${name}!`;

const multiline = `This is line 1.
This is line 2.
This is line 3.`;

4. Destructuring Assignment

Extract data from arrays or objects using a syntax that mirrors the construction of array and object literals.

// Array destructuring
const [a, b, ...rest] = [1, 2, 3, 4, 5];
console.log(a); // 1
console.log(b); // 2
console.log(rest); // [3, 4, 5]

// Object destructuring
const { firstName, lastName, ...otherInfo } = {
  firstName: "John",
  lastName: "Doe",
  age: 30,
  city: "New York"
};
console.log(firstName); // "John"
console.log(lastName); // "Doe"
console.log(otherInfo); // { age: 30, city: "New York" }

5. Default Parameters

Set default values for function parameters.

Before ES6

function greet(name) {
  name = name || "Guest";
  return "Hello, " + name;
}

function createUser(name, age, isAdmin) {
  age = age !== undefined ? age : 25;
  isAdmin = isAdmin !== undefined ? isAdmin : false;
  // ...
}

ES6 Default Parameters

function greet(name = "Guest") {
  return `Hello, ${name}`;
}

function createUser(name, age = 25, isAdmin = false) {
  // ...
}

6. Spread and Rest Operators

The ... syntax serves two purposes: spreading elements and gathering elements.

// Spread operator with arrays
const arr1 = [1, 2, 3];
const arr2 = [...arr1, 4, 5]; // [1, 2, 3, 4, 5]

// Spread operator with objects
const obj1 = { a: 1, b: 2 };
const obj2 = { ...obj1, c: 3 }; // { a: 1, b: 2, c: 3 }

// Rest parameter in functions
function sum(...numbers) {
  return numbers.reduce((total, num) => total + num, 0);
}
console.log(sum(1, 2, 3, 4)); // 10

7. Classes

Syntactic sugar over JavaScript's prototype-based inheritance.

Before ES6

function Person(name, age) {
  this.name = name;
  this.age = age;
}

Person.prototype.greet = function() {
  return "Hi, I'm " + this.name;
};

function Employee(name, age, position) {
  Person.call(this, name, age);
  this.position = position;
}

Employee.prototype = Object.create(Person.prototype);
Employee.prototype.constructor = Employee;

Employee.prototype.getRole = function() {
  return this.position;
};

ES6 Classes

class Person {
  constructor(name, age) {
    this.name = name;
    this.age = age;
  }
  
  greet() {
    return `Hi, I'm ${this.name}`;
  }
}

class Employee extends Person {
  constructor(name, age, position) {
    super(name, age);
    this.position = position;
  }
  
  getRole() {
    return this.position;
  }
}

ES6+ Additional Features

1. Promises and Async/Await (ES6/ES8)

Better ways to handle asynchronous operations.

// Promises (ES6)
function fetchData() {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      resolve("Data received");
    }, 1000);
  });
}

fetchData()
  .then(data => console.log(data))
  .catch(error => console.error(error));

// Async/Await (ES8)
async function getData() {
  try {
    const data = await fetchData();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}

getData();

2. Object Methods (ES6+)

New methods for working with objects.

// Object.assign (ES6)
const target = { a: 1 };
const source = { b: 2, c: 3 };
const result = Object.assign(target, source);
console.log(result); // { a: 1, b: 2, c: 3 }

// Object.entries (ES8)
const obj = { a: 1, b: 2, c: 3 };
console.log(Object.entries(obj)); // [['a', 1], ['b', 2], ['c', 3]]

// Object.values (ES8)
console.log(Object.values(obj)); // [1, 2, 3]

// Object.fromEntries (ES10)
const entries = [['a', 1], ['b', 2], ['c', 3]];
console.log(Object.fromEntries(entries)); // { a: 1, b: 2, c: 3 }

3. Array Methods (ES6+)

New methods for working with arrays.

// Array.from (ES6)
console.log(Array.from('hello')); // ['h', 'e', 'l', 'l', 'o']
console.log(Array.from([1, 2, 3], x => x * 2)); // [2, 4, 6]

// Array.includes (ES7)
const array = [1, 2, 3, 4, 5];
console.log(array.includes(3)); // true
console.log(array.includes(6)); // false

// Array.flat and Array.flatMap (ES10)
const nestedArray = [1, [2, [3, 4]]];
console.log(nestedArray.flat()); // [1, 2, [3, 4]]
console.log(nestedArray.flat(2)); // [1, 2, 3, 4]

const arr = [1, 2, 3];
console.log(arr.flatMap(x => [x, x * 2])); // [1, 2, 2, 4, 3, 6]

4. Optional Chaining (ES11)

Safely access nested object properties without checking each level.

Before ES11

const user = {
  profile: {
    // address might not exist
  }
};

// Verbose property access
const city = user && 
  user.profile && 
  user.profile.address && 
  user.profile.address.city;

With Optional Chaining

const user = {
  profile: {
    // address might not exist
  }
};

// Simplified with optional chaining
const city = user?.profile?.address?.city;

5. Nullish Coalescing (ES11)

Provides a default value only when the left-hand side is null or undefined.

Before ES11

// Problem with || operator
function printMessage(message) {
  // Empty string or 0 would use default
  const text = message || "Default message";
  console.log(text);
}

printMessage("Hello"); // "Hello"
printMessage(""); // "Default message" (not desired)
printMessage(0); // "Default message" (not desired)

With Nullish Coalescing

// Using ?? operator
function printMessage(message) {
  // Only null/undefined use default
  const text = message ?? "Default message";
  console.log(text);
}

printMessage("Hello"); // "Hello"
printMessage(""); // "" (preserves empty string)
printMessage(0); // 0 (preserves zero)

6. BigInt (ES11)

A new primitive type for working with arbitrary precision integers.

// Regular number has limitations
console.log(Number.MAX_SAFE_INTEGER); // 9007199254740991

// BigInt can handle much larger numbers
const bigNumber = 9007199254740991n;
console.log(bigNumber + 1n); // 9007199254740992n
console.log(bigNumber + 2n); // 9007199254740993n

// Can also be created using constructor
const anotherBigNumber = BigInt("9007199254740991");
console.log(anotherBigNumber); // 9007199254740991n

7. Dynamic Import (ES11)

Load modules on demand.

// Static import (loads at parse time)
// import { someFunction } from './module.js';

// Dynamic import (loads on demand)
button.addEventListener('click', async () => {
  try {
    const module = await import('./module.js');
    module.someFunction();
  } catch (error) {
    console.error('Failed to load module:', error);
  }
});
