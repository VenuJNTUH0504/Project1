Variables & Data Types

JavaScript has several data types that are used to store different kinds of values. Understanding these types is fundamental to writing effective JavaScript code.

Primitive Data Types

  • String: Represents textual data. Example: 'Hello', "World"
  • Number: Represents numeric values. Example: 42, 3.14
  • Boolean: Represents logical values. Example: true, false
  • Undefined: Represents a variable that has been declared but not assigned a value. Example: let x;
  • Null: Represents the intentional absence of any object value. Example: let empty = null;
  • Symbol (ES6): Represents a unique identifier. Example: Symbol('description')
  • BigInt (ES2020): Represents integers larger than the Number type can handle. Example: 9007199254740991n

Reference Types

  • Object: Collections of key-value pairs. Example: { name: 'John', age: 30 }
  • Array: Ordered collections of values. Example: [1, 2, 3, 4]
  • Function: Callable objects. Example: function greet() { return 'Hello'; }
  • Date: Represents dates and times. Example: new Date()
  • RegExp: Regular expressions for pattern matching. Example: /pattern/

Variable Declaration

Variables in JavaScript can be declared using:

  • var: Function-scoped, can be redeclared and updated. Not recommended in modern JavaScript.
  • let: Block-scoped, can be updated but not redeclared within the same scope.
  • const: Block-scoped, cannot be updated or redeclared. The value is constant.

Examples

// Variable declarations
let name = 'John';
const age = 30;
var isStudent = true;

// Primitive types
const greeting = 'Hello, World!';  // String
const pi = 3.14159;                // Number
const isActive = false;            // Boolean
let undefinedVar;                  // Undefined
const emptyValue = null;           // Null
const uniqueId = Symbol('id');     // Symbol
const bigNumber = 1234567890123456789012345n; // BigInt

// Reference types
const person = {                   // Object
    firstName: 'John',
    lastName: 'Doe',
    age: 30
};

const numbers = [1, 2, 3, 4, 5];   // Array

function add(a, b) {               // Function
    return a + b;
}

const today = new Date();          // Date

const emailPattern = /^[^\s@]+@[^\s@]+\.[^\s@]+$/; // RegExp

// Type checking
console.log(typeof greeting);      // "string"
console.log(typeof pi);            // "number"
console.log(typeof isActive);      // "boolean"
console.log(typeof undefinedVar);  // "undefined"
console.log(typeof emptyValue);    // "object" (historical bug in JavaScript)
console.log(typeof uniqueId);      // "symbol"
console.log(typeof bigNumber);     // "bigint"
console.log(typeof person);        // "object"
console.log(typeof numbers);       // "object"
console.log(typeof add);           // "function"
console.log(typeof today);         // "object"
console.log(typeof emailPattern);  // "object"

// Dynamic typing
let dynamicVar = 42;
console.log(typeof dynamicVar);    // "number"
dynamicVar = "Now I'm a string";
console.log(typeof dynamicVar);    // "string"

JavaScript is a dynamically typed language, which means variables can change types during runtime. This flexibility can be powerful but also requires careful handling to avoid unexpected behavior.
