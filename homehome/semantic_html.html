Understanding Semantic HTML

What is Semantic HTML?

Semantic HTML refers to using HTML elements that carry meaning about their content, rather than just for presentation. These elements clearly describe their purpose to both the browser and the developer, making the code more readable and accessible.

Why Use Semantic HTML?

  • Better Accessibility: Screen readers can better understand the content structure
  • SEO Benefits: Search engines can better understand your content
  • Easier Maintenance: Code is more organized and easier to maintain
  • Better Mobile Experience: Semantic elements help with responsive design

Common Semantic Elements

<header>

Contains introductory content or navigation links for its parent section.

<header> <h1>Website Title</h1> <nav> <!-- Navigation links --> </nav> </header>

<nav>

Defines a section of navigation links.

<nav> <ul> <li><a href="#">Home</a></li> <li><a href="#">About</a></li> </ul> </nav>

<main>

Contains the main content of the document.

<main> <article> <h2>Article Title</h2> <p>Article content...</p> </article> </main>

<article>

Represents a self-contained composition that can be independently distributable or reusable.

<section>

Defines a thematic grouping of content, typically with a heading.

<aside>

Contains content that is tangentially related to the content around it.

<footer>

Contains footer information for its parent section.

Non-Semantic vs Semantic HTML

Non-Semantic HTML

<div class="header"> <div class="nav"> <div class="menu-item"> </div> </div>

Semantic HTML

<header> <nav> <ul> <li></li> </ul> </nav> </header>

Best Practices

  • Use semantic elements whenever possible instead of generic div elements
  • Ensure proper nesting of semantic elements
  • Use appropriate heading levels (h1-h6) to create a logical document outline
  • Include ARIA labels when necessary for better accessibility
  • Maintain a consistent structure throughout your website

Page Structure Example

<body> <header> <nav> <!-- Navigation content --> </nav> </header> <main> <article> <section> <!-- Article section --> </section> </article> <aside> <!-- Sidebar content --> </aside> </main> <footer> <!-- Footer content --> </footer> </body>
