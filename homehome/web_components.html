Web Components Guide

Introduction to Web Components

Web Components are a set of web platform APIs that allow you to create reusable, encapsulated HTML elements. They consist of three main technologies: Custom Elements, Shadow DOM, and HTML Templates.

Custom Elements

Custom Elements allow you to define your own HTML elements:

class MyComponent extends HTMLElement { constructor() { super(); this.innerHTML = `

My Custom Component

`; } } customElements.define('my-component', MyComponent);

Usage Example:

<my-component></my-component>

Shadow DOM

Shadow DOM provides encapsulation for JavaScript, CSS, and templating in a Web Component:

class ShadowComponent extends HTMLElement { constructor() { super(); const shadow = this.attachShadow({mode: 'open'}); shadow.innerHTML = `
`; } }

HTML Templates

HTML Templates allow you to define reusable markup:

<template id="my-template"> <style> .card { border: 1px solid #ccc; padding: 10px; } </style> <div class="card"> <slot name="title"></slot> <slot name="content"></slot> </div> </template>

Lifecycle Callbacks

  • connectedCallback() - Called when element is added to document
  • disconnectedCallback() - Called when element is removed
  • adoptedCallback() - Called when element is moved to new document
  • attributeChangedCallback() - Called when attributes are changed

Creating a Complete Web Component

class UserCard extends HTMLElement { constructor() { super(); const shadow = this.attachShadow({mode: 'open'}); const template = `

${this.getAttribute('name')}

${this.getAttribute('title')}

`; shadow.innerHTML = template; } } customElements.define('user-card', UserCard);

Using Attributes and Properties

Handling attributes in Web Components:

static get observedAttributes() { return ['name', 'title']; } attributeChangedCallback(name, oldValue, newValue) { if (oldValue !== newValue) { this[name] = newValue; this.render(); } }

Event Handling

Adding event listeners to Web Components:

connectedCallback() { this.shadowRoot.querySelector('button') .addEventListener('click', () => { this.dispatchEvent(new CustomEvent('custom-click')); }); }

Best Practices

  • Use Shadow DOM for style encapsulation
  • Follow naming conventions (must contain a hyphen)
  • Keep components focused and single-purpose
  • Document your components well
  • Handle cleanup in disconnectedCallback
  • Use attributes for configuration

Browser Support and Polyfills

For older browsers, you may need polyfills:

  • @webcomponents/webcomponentsjs
  • Custom Elements polyfill
  • Shadow DOM polyfill
  • HTML Template polyfill

Performance Considerations

  • Minimize Shadow DOM operations
  • Use template elements for better performance
  • Lazy load components when possible
  • Optimize attribute observations
  • Consider using adoptedStyleSheets for shared styles
