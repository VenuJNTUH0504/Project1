HTML Accessibility Guide

Introduction to Web Accessibility

Web accessibility ensures that websites and web applications are usable by people with disabilities. This includes visual, auditory, physical, speech, cognitive, and neurological disabilities. Making your website accessible is not just good practice—it's often a legal requirement.

Semantic HTML

Using semantic HTML elements helps screen readers and other assistive technologies understand your content:

<header> <nav> <ul> <li><a href="#">Home</a></li> </ul> </nav> </header> <main> <article> <h1>Main Content</h1> <section> <h2>Section Title</h2> </section> </article> </main> <footer> <p>Footer Content</p> </footer>

ARIA Roles and Attributes

ARIA (Accessible Rich Internet Applications) provides additional information to assistive technologies:

<button role="button" aria-label="Close dialog" aria-pressed="false"> × </button> <div role="alert" aria-live="polite"> Form submitted successfully! </div>

Image Accessibility

Always provide alternative text for images:

<img src="logo.png" alt="Company Logo"> <img src="decorative.png" alt=""> <figure> <img src="chart.png" alt="Sales chart showing growth"> <figcaption>Annual sales growth 2020-2023</figcaption> </figure>

Keyboard Navigation

  • Ensure all interactive elements are focusable
  • Maintain a logical tab order
  • Provide visible focus indicators
  • Implement skip links for main content
  • Support keyboard shortcuts where appropriate

Color and Contrast

Ensure sufficient color contrast and don't rely solely on color to convey information:

  • Minimum contrast ratio of 4.5:1 for normal text
  • Minimum contrast ratio of 3:1 for large text
  • Provide alternative indicators besides color
  • Test with color blindness simulators

Multimedia Accessibility

<video controls> <source src="video.mp4" type="video/mp4"> <track kind="captions" src="captions.vtt" srclang="en"> <track kind="descriptions" src="descriptions.vtt" srclang="en"> </video>

Common ARIA Roles

Role Description
banner Site header content
navigation Navigation links
main Main content area
complementary Supporting content
contentinfo Footer information

Testing Accessibility

Tools and Methods:

  • Screen readers (NVDA, VoiceOver, JAWS)
  • Keyboard navigation testing
  • WAVE Web Accessibility Tool
  • Lighthouse Accessibility Audit
  • Color contrast analyzers
  • Manual testing with assistive technologies

Best Practices

  • Use semantic HTML elements
  • Provide alternative text for images
  • Ensure keyboard accessibility
  • Maintain sufficient color contrast
  • Use ARIA attributes appropriately
  • Test with various assistive technologies
  • Include proper heading hierarchy
  • Provide captions and transcripts for multimedia
