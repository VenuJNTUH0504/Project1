Union Types in TypeScript

Union types are a powerful feature in TypeScript that allows a variable to hold values of multiple types. They provide flexibility while maintaining type safety, letting you express a value that could be one of several types.

Basic Union Types

A union type is written as a vertical bar (|) separated list of types:

// Variable that can be either a string or a number
let id: string | number;

// Both assignments are valid
id = 101;
id = "U101";

// Error: Type 'boolean' is not assignable to type 'string | number'
// id = true;

Working with Union Types

When working with union types, you can only access members that are common to all types in the union:

function printId(id: number | string) {
    console.log("ID: " + id);  // OK: toString() is available on both string and number
    
    // Error: Property 'toUpperCase' does not exist on type 'number | string'.
    // Property 'toUpperCase' does not exist on type 'number'.
    // console.log(id.toUpperCase());
}

Type Narrowing

To work with properties that exist on only one type in the union, you need to narrow the type using type guards:

function printId(id: number | string) {
    if (typeof id === "string") {
        // In this branch, id is of type 'string'
        console.log(id.toUpperCase());
    } else {
        // In this branch, id is of type 'number'
        console.log(id.toFixed(2));
    }
}

Common Type Guards

TypeScript supports several ways to narrow types:

  • typeof for primitive types
  • instanceof for class instances
  • Custom type predicates with user-defined type guards
  • The in operator for checking if a property exists on an object

Union Types with Arrays

You can create arrays that contain elements of different types using union types:

// Array that can contain both strings and numbers
let mixed: (string | number)[] = ["hello", 42, "world", 100];

// Array that can be either all strings or all numbers
let stringOrNumberArray: string[] | number[] = [1, 2, 3];
stringOrNumberArray = ["a", "b", "c"];  // Also valid

// Error: Type '(string | number)[]' is not assignable to type 'string[] | number[]'
// stringOrNumberArray = [1, "a", 2];  // Not valid

Literal Types in Unions

Union types can include literal types, which are specific values rather than general types:

// Function that accepts specific string values
function setAlignment(alignment: "left" | "right" | "center") {
    // ...
}

setAlignment("left");  // OK
setAlignment("right");  // OK
// Error: Argument of type '"top"' is not assignable to parameter of type '"left" | "right" | "center"'
// setAlignment("top");

Discriminated Unions

A common pattern in TypeScript is to create a discriminated union, where each type in the union contains a common property (the "discriminant") that can be used to narrow the type:

interface Square {
    kind: "square";
    size: number;
}

interface Rectangle {
    kind: "rectangle";
    width: number;
    height: number;
}

interface Circle {
    kind: "circle";
    radius: number;
}

type Shape = Square | Rectangle | Circle;

function calculateArea(shape: Shape): number {
    switch (shape.kind) {
        case "square":
            return shape.size * shape.size;
        case "rectangle":
            return shape.width * shape.height;
        case "circle":
            return Math.PI * shape.radius * shape.radius;
    }
}

Exhaustiveness Checking

TypeScript can help ensure you've handled all possible types in a union by using a never type and the compiler's exhaustiveness checking:

function assertNever(x: never): never {
    throw new Error("Unexpected object: " + x);
}

function calculateArea(shape: Shape): number {
    switch (shape.kind) {
        case "square":
            return shape.size * shape.size;
        case "rectangle":
            return shape.width * shape.height;
        case "circle":
            return Math.PI * shape.radius * shape.radius;
        default:
            // If a new shape type is added and not handled above,
            // TypeScript will generate a compile-time error
            return assertNever(shape);
    }
}

Intersection Types vs Union Types

Union Types (|)

A value can be one of the specified types

type A = string | number;
// Value can be either string OR number

Intersection Types (&)

A value must satisfy all of the specified types

type B = { name: string } & { age: number };
// Value must have both name AND age

Optional Properties vs Union with Undefined

There are two ways to express that a property might not be present:

// Using optional property
interface User1 {
    name: string;
    email?: string;  // email might not be present
}

// Using union with undefined
interface User2 {
    name: string;
    email: string | undefined;  // email must be present, but might be undefined
}

// These are different:
const user1: User1 = { name: "Alice" };  // OK, email can be omitted
const user2: User2 = { name: "Bob" };    // Error: Property 'email' is missing
const user3: User2 = { name: "Charlie", email: undefined };  // OK
