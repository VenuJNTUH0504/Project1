TypeScript Type Inference

Understanding how TypeScript automatically determines types

What is Type Inference?

Type inference is a feature in TypeScript that automatically determines the type of a variable based on its value. This means you don't always have to explicitly declare types, as TypeScript can often figure them out for you.

TypeScript's type inference helps reduce the amount of type annotations you need to write while still maintaining type safety.

// Type inference in variable declarations
let message = "Hello, World!";  // TypeScript infers type as string
let count = 42;                 // TypeScript infers type as number
let isActive = true;            // TypeScript infers type as boolean

// If you try to assign a different type later, TypeScript will show an error
// message = 100;  // Error: Type 'number' is not assignable to type 'string'

Inference in Function Return Types

TypeScript can also infer the return type of functions based on the return statements:

// Return type is inferred as number
function add(a: number, b: number) {
    return a + b;
}

// Return type is inferred as string
function greet(name: string) {
    return `Hello, ${name}!`;
}

// TypeScript infers the correct return types
let sum = add(5, 3);         // sum is inferred as number
let greeting = greet("John"); // greeting is inferred as string

Contextual Typing

TypeScript can infer types based on the context in which expressions are used. This is called contextual typing:

// The type of 'e' is inferred from the context (MouseEvent)
document.addEventListener("click", function(e) {
    console.log(e.button);  // TypeScript knows e is a MouseEvent
});

// TypeScript infers the type of the parameter in arrow functions too
const names = ["Alice", "Bob", "Charlie"];
names.forEach(name => {
    console.log(name.toUpperCase());  // TypeScript knows 'name' is a string
});

Type Inference with Objects and Arrays

TypeScript infers types for object literals and arrays:

// Object type inference
let person = {
    name: "Alice",
    age: 30,
    isEmployed: true
};
// TypeScript infers: { name: string; age: number; isEmployed: boolean; }

// Array type inference
let numbers = [1, 2, 3, 4, 5];
// TypeScript infers: number[]

let mixed = [1, "two", 3, "four"];
// TypeScript infers: (string | number)[]

Best Practices with Type Inference

While type inference is powerful, there are times when explicit type annotations are better:

// When you want a wider type than what would be inferred
let id: number | string = 42;  // Without annotation, id would be just number

// When you want to document your code for other developers
function calculateTotal(items: CartItem[]): number {
    return items.reduce((total, item) => total + item.price, 0);
}

// When TypeScript might infer a type that's too specific
const config: Configuration = {
    endpoint: "api/data",
    timeout: 3000
};

// When you're declaring a variable without initializing it
let result: number;  // Without this, result would be 'any'
// ... later ...
result = calculateSomething();

Practical Examples

// Type inference with destructuring
const [first, second] = [1, 2];
// first and second are inferred as numbers

const { name, age } = { name: "Alice", age: 30, country: "USA" };
// name is inferred as string, age as number

// Type inference with generics
function identity(arg: T): T {
    return arg;
}

let output = identity("myString");
// output is inferred as string

// Type inference with async/await
async function fetchData() {
    const response = await fetch('https://api.example.com/data');
    const data = await response.json();
    return data;
}
// Return type is inferred as Promise
