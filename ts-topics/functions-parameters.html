Functions & Parameters in TypeScript

Functions are the fundamental building blocks of any application in TypeScript. TypeScript enhances JavaScript functions with type annotations for parameters and return values, making your code more robust and maintainable.

Function Declarations

There are several ways to declare functions in TypeScript:

Named Functions

// Named function with type annotations
function add(x: number, y: number): number {
    return x + y;
}

// Call the function
let result = add(5, 3);  // result: 8

Anonymous Functions

// Anonymous function assigned to a variable
let add = function(x: number, y: number): number {
    return x + y;
};

// Call the function
let result = add(5, 3);  // result: 8

Arrow Functions

// Arrow function with type annotations
let add = (x: number, y: number): number => {
    return x + y;
};

// Shorter syntax for simple return
let multiply = (x: number, y: number): number => x * y;

// Call the functions
let sum = add(5, 3);       // sum: 8
let product = multiply(5, 3);  // product: 15

Function Type Annotations

TypeScript allows you to specify the types of function parameters and return values:

// Function with parameter and return type annotations
function greet(name: string): string {
    return `Hello, ${name}!`;
}

// Function type annotation for a variable
let sayHello: (name: string) => string;

// Assign a compatible function
sayHello = function(name: string) {
    return `Hello, ${name}!`;
};

// Alternative arrow function syntax
sayHello = (name: string) => `Hello, ${name}!`;

Type Inference for Return Types

TypeScript can often infer the return type of a function, but it's a good practice to explicitly specify it for better documentation and to catch errors if the implementation changes:

// Return type is inferred as number
function add(x: number, y: number) {
    return x + y;
}

// Explicitly specified return type
function add(x: number, y: number): number {
    return x + y;
}

Optional Parameters

In TypeScript, you can make function parameters optional by adding a question mark (?) after the parameter name:

// Optional parameter (lastName is optional)
function buildName(firstName: string, lastName?: string): string {
    if (lastName) {
        return `${firstName} ${lastName}`;
    } else {
        return firstName;
    }
}

// Both calls are valid
let result1 = buildName("John");          // "John"
let result2 = buildName("John", "Doe");   // "John Doe"

Important Note

Optional parameters must come after required parameters. The following is not allowed:

// Error: Required parameter cannot follow an optional parameter
function buildName(firstName?: string, lastName: string): string {
    // ...
}

Default Parameters

You can provide default values for parameters, which will be used if the parameter is not provided or is undefined:

// Default parameter (greeting has a default value)
function greet(name: string, greeting: string = "Hello"): string {
    return `${greeting}, ${name}!`;
}

// Both calls are valid
let result1 = greet("John");              // "Hello, John!"
let result2 = greet("John", "Hi");        // "Hi, John!"
let result3 = greet("John", undefined);   // "Hello, John!"

Rest Parameters

Rest parameters allow you to work with multiple parameters as a group. In TypeScript, you can type rest parameters as arrays:

// Rest parameter (items is an array of strings)
function buildList(firstItem: string, ...restItems: string[]): string {
    return firstItem + " " + restItems.join(" ");
}

// Call with multiple arguments
let list = buildList("Apple", "Banana", "Cherry");  // "Apple Banana Cherry"

Function Overloads

TypeScript allows you to specify multiple function types for the same function, known as function overloads. This is useful when a function can accept different types of arguments:

// Function overload signatures
function getItem(id: number): Item;
function getItem(name: string): Item;

// Implementation signature (must be compatible with all overloads)
function getItem(idOrName: number | string): Item {
    if (typeof idOrName === "number") {
        // Find item by id
        return database.getItemById(idOrName);
    } else {
        // Find item by name
        return database.getItemByName(idOrName);
    }
}

Overload Resolution

When TypeScript resolves overloads, it picks the first matching overload going from top to bottom. Put more specific overloads before less specific ones.

Generic Functions

Generic functions allow you to create functions that work with a variety of types rather than a single type:

// Generic function
function identity<T>(arg: T): T {
    return arg;
}

// Call with explicit type argument
let output1 = identity<string>("hello");  // Type: string

// Call with inferred type argument
let output2 = identity(42);  // Type: number

Generic functions are particularly useful for creating reusable components:

// Generic function with constraints
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
    return obj[key];
}

let person = { name: "John", age: 30 };
let name = getProperty(person, "name");  // Type: string
let age = getProperty(person, "age");    // Type: number
// let invalid = getProperty(person, "job");  // Error: "job" is not in keyof person

Contextual Typing

TypeScript can infer the types of function parameters based on the context in which the function is used:

// Array.forEach provides context for the callback function
let names = ["Alice", "Bob", "Charlie"];

// Parameter 'name' is inferred to be of type string
names.forEach(function(name) {
    console.log(name.toUpperCase());  // No error, TypeScript knows 'name' is a string
});

Void Type

The void type is used to indicate that a function doesn't return a value:

// Function with void return type
function logMessage(message: string): void {
    console.log(message);
    // No return statement needed
}

// Even if there is a return statement with no value, the return type is void
function doSomething(): void {
    // Do something...
    return;  // Valid
    // return "value";  // Error: Type 'string' is not assignable to type 'void'
}

Never Type

The never type represents values that never occur. It is used for functions that never return (e.g., functions that throw exceptions or have infinite loops):

// Function that always throws an exception
function throwError(message: string): never {
    throw new Error(message);
}

// Function with an infinite loop
function infiniteLoop(): never {
    while (true) {
        // Some code...
    }
}

this Parameters

TypeScript allows you to specify the type of this in function calls:

interface User {
    id: number;
    name: string;
    updateName(newName: string): void;
}

function updateUser(this: User, newName: string): void {
    this.name = newName;
}

let user: User = {
    id: 1,
    name: "John",
    updateName: updateUser
};

user.updateName("Jane");  // Works
// updateUser("Jane");  // Error: 'this' context is required

Function Type Compatibility

TypeScript uses structural typing for function compatibility. A function is compatible with another if its parameters are compatible (contravariant) and its return type is compatible (covariant):

// Function type compatibility
let func1 = (a: number) => 0;
let func2 = (b: number, s: string) => 0;

// func2 can be assigned to func1, but not vice versa
func1 = func2;  // Error: func2 has more required parameters
func2 = func1;  // OK: Extra parameters are ignored when assigning to func2
