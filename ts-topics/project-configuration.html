TypeScript Project Configuration

Configuring a TypeScript project properly is essential for a smooth development experience. TypeScript offers various configuration options that allow you to customize the compiler's behavior, specify project structure, and enforce coding standards.

tsconfig.json

The tsconfig.json file is the heart of TypeScript project configuration. It specifies the root files and compiler options required to compile the project:

// Basic tsconfig.json
{
  "compilerOptions": {
    "target": "es2020",
    "module": "commonjs",
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true,
    "outDir": "./dist"
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "**/*.spec.ts"]
}

Creating a tsconfig.json

You can generate a basic tsconfig.json file using the TypeScript compiler:

// Initialize a TypeScript project
npx tsc --init

This will create a tsconfig.json file with default settings and helpful comments.

Key Configuration Sections

compilerOptions

The most important section of tsconfig.json, containing settings that control how TypeScript compiles your code:

Basic Compiler Options

"compilerOptions": {
  // JavaScript target version
  "target": "es2020",        // Specify ECMAScript target version
  
  // Module system
  "module": "commonjs",      // Specify module code generation
  
  // Output configuration
  "outDir": "./dist",        // Redirect output to this directory
  "rootDir": "./src",        // Specify the root directory of input files
  
  // Type checking
  "strict": true,            // Enable all strict type-checking options
  
  // Module resolution
  "moduleResolution": "node", // Specify module resolution strategy
  "esModuleInterop": true,    // Enable interoperability between CommonJS and ES Modules
  
  // Source maps
  "sourceMap": true,         // Generate source maps for debugging
}

Type Checking Options

"compilerOptions": {
  // Individual strict checks (all enabled when strict: true)
  "noImplicitAny": true,           // Raise error on expressions with implied 'any' type
  "strictNullChecks": true,        // Enable strict null checks
  "strictFunctionTypes": true,     // Enable strict checking of function types
  "strictBindCallApply": true,     // Enable strict 'bind', 'call', and 'apply' methods
  "strictPropertyInitialization": true, // Ensure class properties are initialized
  "noImplicitThis": true,          // Raise error on 'this' expressions with implied 'any' type
  "alwaysStrict": true,            // Parse in strict mode and emit "use strict"
  
  // Additional checks
  "noUnusedLocals": true,          // Report errors on unused locals
  "noUnusedParameters": true,      // Report errors on unused parameters
  "noImplicitReturns": true,       // Report error when not all code paths return a value
  "noFallthroughCasesInSwitch": true, // Report errors for fallthrough cases in switch
  "noUncheckedIndexedAccess": true // Add 'undefined' to indexed access results
}

Module and Path Resolution

"compilerOptions": {
  // Module resolution
  "baseUrl": "./",                 // Base directory to resolve non-relative module names
  "paths": {                       // Path mapping for module imports
    "@app/*": ["src/app/*"],
    "@shared/*": ["src/shared/*"],
    "@models/*": ["src/models/*"]
  },
  "rootDirs": [                    // List of root folders whose combined content represent the structure of the project at runtime
    "./src",
    "./tests"
  ],
  
  // Type definitions
  "typeRoots": [                   // List of folders to include type definitions from
    "./node_modules/@types",
    "./typings"
  ],
  "types": [                       // Type declaration files to be included in compilation
    "node",
    "jest",
    "express"
  ]
}

Output and Declaration Options

"compilerOptions": {
  // JavaScript output
  "outDir": "./dist",              // Output directory
  "outFile": "./dist/bundle.js",   // Concatenate and emit output to single file (only with AMD/System modules)
  "removeComments": true,          // Remove comments in output
  "noEmit": false,                 // Do not emit outputs
  "noEmitOnError": true,           // Do not emit outputs if any errors were reported
  
  // Declaration files
  "declaration": true,             // Generate .d.ts files
  "declarationDir": "./types",     // Output directory for generated declaration files
  "declarationMap": true,          // Generate sourcemaps for d.ts files
  
  // Source maps
  "sourceMap": true,               // Generate source maps
  "inlineSourceMap": false,        // Emit a single file with source maps instead of separate files
  "inlineSources": false           // Emit the source alongside the sourcemaps
}

include and exclude

These options specify which files should be included or excluded from the compilation:

// File selection
{
  "include": [
    "src/**/*"                     // Include all files in src directory
  ],
  "exclude": [
    "node_modules",                // Exclude node_modules
    "**/*.spec.ts",                // Exclude test files
    "src/temp/**/*"                // Exclude temporary files
  ]
}

files

Explicitly list files to be included in the compilation (alternative to include):

// Explicit file list
{
  "files": [
    "src/main.ts",
    "src/app.ts",
    "src/types.d.ts"
  ]
}

extends

Inherit configurations from another file:

// Extending configurations
{
  "extends": "./tsconfig.base.json",
  "compilerOptions": {
    // Override specific options
    "outDir": "./dist/dev"
  }
}

references

For project references in a monorepo setup:

// Project references
{
  "compilerOptions": {
    "composite": true,             // Enable project references
    "declaration": true,
    "outDir": "./dist"
  },
  "references": [
    { "path": "../common" },       // Reference to another project
    { "path": "../api" }
  ]
}

Common Configuration Presets

Node.js Application

// tsconfig.json for Node.js
{
  "compilerOptions": {
    "target": "es2020",
    "module": "commonjs",
    "lib": ["es2020"],
    "sourceMap": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "moduleResolution": "node",
    "esModuleInterop": true,
    "resolveJsonModule": true,
    "types": ["node"],
    "typeRoots": ["./node_modules/@types", "./src/types"]
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "**/*.spec.ts"]
}

React Application

// tsconfig.json for React
{
  "compilerOptions": {
    "target": "es2020",
    "lib": ["dom", "dom.iterable", "esnext"],
    "allowJs": true,
    "skipLibCheck": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "forceConsistentCasingInFileNames": true,
    "noFallthroughCasesInSwitch": true,
    "module": "esnext",
    "moduleResolution": "node",
    "resolveJsonModule": true,
    "isolatedModules": true,
    "noEmit": true,
    "jsx": "react-jsx",
    "baseUrl": "./src",
    "paths": {
      "@components/*": ["components/*"],
      "@hooks/*": ["hooks/*"],
      "@utils/*": ["utils/*"]
    }
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules"]
}

Library Package

// tsconfig.json for a library
{
  "compilerOptions": {
    "target": "es2018",
    "module": "esnext",
    "lib": ["es2018", "dom"],
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "outDir": "./dist",
    "rootDir": "./src",
    "strict": true,
    "moduleResolution": "node",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  },
  "include": ["src/**/*"],
  "exclude": ["node_modules", "**/*.spec.ts", "**/*.test.ts"]
}

Project Organization

Monorepo Setup

For larger projects, you might use a monorepo structure with multiple tsconfig files:

// Root tsconfig.json
{
  "files": [],
  "references": [
    { "path": "packages/common" },
    { "path": "packages/server" },
    { "path": "packages/client" }
  ]
}

// packages/common/tsconfig.json
{
  "compilerOptions": {
    "composite": true,
    "outDir": "./dist",
    "rootDir": "./src",
    // ...other options
  },
  "include": ["src/**/*"]
}

// packages/server/tsconfig.json
{
  "compilerOptions": {
    "composite": true,
    "outDir": "./dist",
    "rootDir": "./src",
    // ...other options
  },
  "references": [
    { "path": "../common" }
  ],
  "include": ["src/**/*"]
}

Path Aliases

Configure path aliases to avoid deep relative imports:

// tsconfig.json with path aliases
{
  "compilerOptions": {
    "baseUrl": "./",
    "paths": {
      "@app/*": ["src/app/*"],
      "@shared/*": ["src/shared/*"],
      "@utils/*": ["src/utils/*"],
      "@models/*": ["src/models/*"]
    }
    // ...other options
  }
}

// Usage in code
import { User } from '@models/user';
import { formatDate } from '@utils/date';
import { Button } from '@shared/components/button';

Path Aliases with Bundlers

When using path aliases, you may need additional configuration in your bundler:

  • For webpack, use the resolve.alias option
  • For Jest, configure moduleNameMapper
  • For ESLint, use eslint-import-resolver-typescript

Type Checking Configuration

Strict Mode

Enabling strict mode activates TypeScript's strictest type checking options:

// Strict mode configuration
{
  "compilerOptions": {
    "strict": true,
    // This enables all of these:
    // "noImplicitAny": true,
    // "strictNullChecks": true,
    // "strictFunctionTypes": true,
    // "strictBindCallApply": true,
    // "strictPropertyInitialization": true,
    // "noImplicitThis": true,
    // "alwaysStrict": true,
    
    // You can override individual options:
    "strictNullChecks": false  // Disable just this check
  }
}

Additional Checks

Beyond strict mode, TypeScript offers additional checks to catch common issues:

// Additional type checking options
{
  "compilerOptions": {
    "noUnusedLocals": true,           // Error on unused local variables
    "noUnusedParameters": true,        // Error on unused function parameters
    "noImplicitReturns": true,         // Error when not all code paths return a value
    "noFallthroughCasesInSwitch": true, // Error on fallthrough cases in switch
    "noUncheckedIndexedAccess": true,  // Add undefined to indexed access
    "noPropertyAccessFromIndexSignature": true, // Error when accessing properties via string index
    "exactOptionalPropertyTypes": true // Enforce exact optional property types
  }
}

Module Resolution

Module Systems

// Module system configuration
{
  "compilerOptions": {
    "module": "commonjs",     // CommonJS (Node.js)
    // Other options: "esnext", "amd", "system", "umd", "es2015", "es2020"
    
    "moduleResolution": "node",
    // Other option: "classic" (legacy)
    
    "esModuleInterop": true,  // Allow default imports from CommonJS modules
    "allowSyntheticDefaultImports": true  // Allow default imports without actual exports
  }
}

Type Definition Resolution

// Type definition resolution
{
  "compilerOptions": {
    "typeRoots": [
      "./node_modules/@types",
      "./typings"
    ],
    "types": [
      "node",
      "jest",
      "express"
    ]
  }
}

Compiler Plugins and Custom Transformers

TypeScript supports plugins that can transform code during compilation:

// Using ts-node with a transformer plugin
// tsconfig.json
{
  "compilerOptions": {
    // ...standard options
  },
  "ts-node": {
    "transpileOnly": true,
    "transpiler": "ts-node/transpilers/swc"
  }
}

// Using ttypescript with transformers
// tsconfig.json
{
  "compilerOptions": {
    "plugins": [
      { "transform": "typescript-transform-paths" },
      { "transform": "typescript-transform-imports" }
    ]
  }
}

Integration with Other Tools

ESLint

// .eslintrc.js
module.exports = {
  parser: '@typescript-eslint/parser',
  parserOptions: {
    project: './tsconfig.json',
    tsconfigRootDir: __dirname,
  },
  plugins: ['@typescript-eslint'],
  extends: [
    'eslint:recommended',
    'plugin:@typescript-eslint/recommended',
    'plugin:@typescript-eslint/recommended-requiring-type-checking',
  ],
  rules: {
    // Custom rules
  }
};

Jest

// jest.config.js
module.exports = {
  preset: 'ts-jest',
  testEnvironment: 'node',
  moduleNameMapper: {
    '^@app/(.*)$': '<rootDir>/src/app/$1',
    '^@shared/(.*)$': '<rootDir>/src/shared/$1',
  },
};

Webpack

// webpack.config.js
const path = require('path');

module.exports = {
  entry: './src/index.ts',
  module: {
    rules: [
      {
        test: /\.tsx?$/,
        use: 'ts-loader',
        exclude: /node_modules/,
      },
    ],
  },
  resolve: {
    extensions: ['.tsx', '.ts', '.js'],
    alias: {
      '@app': path.resolve(__dirname, 'src/app'),
      '@shared': path.resolve(__dirname, 'src/shared'),
    },
  },
  output: {
    filename: 'bundle.js',
    path: path.resolve(__dirname, 'dist'),
  },
};

Performance Optimization

Incremental Compilation

// Incremental compilation
{
  "compilerOptions": {
    "incremental": true,           // Enable incremental compilation
    "tsBuildInfoFile": "./buildcache/tsconfig.tsbuildinfo"  // Where to store the incremental build info
  }
}

Type Checking Optimization

// Type checking optimization
{
  "compilerOptions": {
    "skipLibCheck": true,          // Skip type checking of declaration files
    "skipDefaultLibCheck": true    // Skip type checking of default library declaration files
  }
}

Build Mode vs. Watch Mode

// Build commands
// One-time build
tsc -p tsconfig.json

// Watch mode (continuous compilation)
tsc -p tsconfig.json --watch

// Build with project references
tsc -b                             // Build all referenced projects
tsc -b --verbose                   // Build with detailed logging
tsc -b --clean                     // Clean output of all projects
