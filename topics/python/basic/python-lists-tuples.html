Python Lists and Tuples

Understanding Python's sequence data types

Lists

Lists are mutable sequences of items:

# Creating lists
numbers = [1, 2, 3, 4, 5]
fruits = ["apple", "banana", "cherry"]
mixed = [1, "hello", 3.14, True]

# Accessing elements
print(numbers[0])      # Output: 1
print(fruits[-1])      # Output: cherry
print(mixed[1:3])      # Output: ["hello", 3.14]

# Modifying lists
fruits[1] = "orange"   # Change element
fruits.append("grape") # Add element
fruits.remove("apple") # Remove element
fruits.insert(1, "kiwi") # Insert element

List Methods

Common list operations and methods:

# List methods
numbers = [3, 1, 4, 1, 5, 9, 2, 6, 5]

numbers.sort()          # Sort in place
numbers.reverse()       # Reverse in place
count = numbers.count(5) # Count occurrences
index = numbers.index(4) # Find index

# List operations
combined = numbers + [7, 8, 9]  # Concatenation
repeated = [1, 2] * 3          # Repetition
length = len(numbers)           # Length

# List comprehension
squares = [x**2 for x in range(10)]
evens = [x for x in range(20) if x % 2 == 0]

Tuples

Tuples are immutable sequences:

# Creating tuples
coordinates = (10, 20)
colors = ("red", "green", "blue")
single = (42,)  # Note the comma for single-element tuple

# Accessing elements
print(coordinates[0])  # Output: 10
print(colors[-1])      # Output: blue
print(colors[1:])      # Output: ("green", "blue")

# Tuple unpacking
x, y = coordinates
r, g, b = colors

# Tuple operations
combined = coordinates + (30, 40)
repeated = colors * 2
length = len(colors)

Lists vs Tuples

Key differences between lists and tuples:

  • Lists are mutable, tuples are immutable
  • Lists use square brackets [], tuples use parentheses ()
  • Lists have more built-in methods
  • Tuples are generally faster than lists
  • Tuples can be used as dictionary keys, lists cannot
  • Tuples are often used for heterogeneous data, lists for homogeneous data

Nested Sequences

Working with nested lists and tuples:

# Nested lists
matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# Accessing nested elements
print(matrix[1][2])  # Output: 6

# Nested tuples
points = ((1, 2), (3, 4), (5, 6))

# List of tuples
student_grades = [
    ("Alice", 85),
    ("Bob", 92),
    ("Charlie", 78)
]

# Dictionary with tuple keys
coordinate_map = {
    (0, 0): "origin",
    (1, 1): "point one"
}

Best Practices

  • Use lists when you need to modify the sequence
  • Use tuples for fixed collections of items
  • Prefer list comprehensions over loops when possible
  • Use meaningful variable names for sequences
  • Consider using tuples for function return values
  • Use appropriate sequence types for the data
  • Document complex nested structures
