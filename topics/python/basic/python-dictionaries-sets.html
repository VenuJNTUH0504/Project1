Python Dictionaries and Sets

Understanding Python's mapping and set data types

Dictionaries

Dictionaries are mutable mappings of key-value pairs:

# Creating dictionaries
person = {
    "name": "John",
    "age": 30,
    "city": "New York"
}

# Accessing values
print(person["name"])  # Output: John
print(person.get("age"))  # Output: 30

# Modifying dictionaries
person["age"] = 31  # Update value
person["job"] = "Engineer"  # Add new key-value pair
del person["city"]  # Remove key-value pair

# Dictionary methods
keys = person.keys()  # Get all keys
values = person.values()  # Get all values
items = person.items()  # Get all key-value pairs

Dictionary Methods

Common dictionary operations and methods:

# Dictionary methods
person = {"name": "John", "age": 30}

# Checking existence
if "name" in person:
    print("Name exists")

# Getting with default
age = person.get("age", 0)  # Returns 0 if key doesn't exist

# Updating dictionary
person.update({"city": "New York", "job": "Engineer"})

# Removing items
person.pop("age")  # Remove and return value
person.popitem()  # Remove and return last item
person.clear()  # Remove all items

# Dictionary comprehension
squares = {x: x**2 for x in range(5)}
even_squares = {x: x**2 for x in range(10) if x % 2 == 0}

Sets

Sets are mutable collections of unique elements:

# Creating sets
fruits = {"apple", "banana", "cherry"}
numbers = {1, 2, 3, 4, 5}

# Adding elements
fruits.add("orange")
fruits.update(["grape", "kiwi"])

# Removing elements
fruits.remove("apple")  # Raises error if not found
fruits.discard("banana")  # No error if not found
fruits.pop()  # Remove and return random element

# Set operations
set1 = {1, 2, 3}
set2 = {3, 4, 5}

union = set1 | set2  # {1, 2, 3, 4, 5}
intersection = set1 & set2  # {3}
difference = set1 - set2  # {1, 2}
symmetric_diff = set1 ^ set2  # {1, 2, 4, 5}

Set Methods

Common set operations and methods:

# Set methods
set1 = {1, 2, 3}
set2 = {3, 4, 5}

# Set operations
set1.union(set2)  # Returns new set with all elements
set1.intersection(set2)  # Returns new set with common elements
set1.difference(set2)  # Returns new set with elements in set1 but not set2
set1.symmetric_difference(set2)  # Returns new set with elements in either set but not both

# Set comparisons
set1.issubset(set2)  # True if all elements of set1 are in set2
set1.issuperset(set2)  # True if all elements of set2 are in set1
set1.isdisjoint(set2)  # True if sets have no common elements

# Set comprehension
squares = {x**2 for x in range(10)}
evens = {x for x in range(20) if x % 2 == 0}

Dictionaries vs Sets

Key differences between dictionaries and sets:

  • Dictionaries store key-value pairs, sets store unique elements
  • Dictionaries use curly braces with colons {key: value}, sets use curly braces {element}
  • Dictionaries are ordered (Python 3.7+), sets are unordered
  • Dictionaries are mutable, sets are mutable
  • Dictionaries are used for mapping relationships, sets for unique collections
  • Both use hash tables for efficient lookups

Nested Structures

Working with nested dictionaries and sets:

# Nested dictionaries
company = {
    "name": "Tech Corp",
    "departments": {
        "engineering": {"head": "John", "size": 50},
        "marketing": {"head": "Sarah", "size": 20}
    }
}

# Accessing nested values
print(company["departments"]["engineering"]["head"])  # Output: John

# Dictionary of sets
student_courses = {
    "Alice": {"Math", "Physics", "Chemistry"},
    "Bob": {"History", "English", "Math"}
}

# Set of dictionaries
unique_people = {
    {"name": "John", "age": 30},
    {"name": "Sarah", "age": 25}
}

Best Practices

  • Use dictionaries for key-value relationships
  • Use sets for unique collections and set operations
  • Choose appropriate keys for dictionaries
  • Use dictionary comprehension for transformations
  • Consider using sets for membership testing
  • Document complex nested structures
  • Use appropriate data structures for the task
