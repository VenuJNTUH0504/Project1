Exception Handling in Java

Understanding how to handle errors and exceptions in Java programs

Introduction to Exceptions

Exceptions are events that occur during the execution of a program that disrupt the normal flow of instructions.

Types of exceptions:

  • Checked Exceptions (Compile-time exceptions)
  • Unchecked Exceptions (Runtime exceptions)
  • Errors (Serious problems that are not expected to be caught)

Try-Catch Block

The try-catch block is used to handle exceptions in Java.

Example: Basic Try-Catch

try {
    // Code that might throw an exception
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Handle the exception
    System.out.println("Cannot divide by zero");
}

Key points about try-catch:

  • Try block contains code that might throw an exception
  • Catch block handles the exception
  • Multiple catch blocks can be used

Finally Block

The finally block is used to execute important code such as closing resources, regardless of whether an exception occurs or not.

Example: Try-Catch-Finally

try {
    // Code that might throw an exception
    FileReader file = new FileReader("test.txt");
} catch (FileNotFoundException e) {
    System.out.println("File not found");
} finally {
    // This code will always execute
    System.out.println("Finally block executed");
}

Throw and Throws

The 'throw' keyword is used to explicitly throw an exception, while 'throws' is used to declare exceptions that might occur in a method.

Example: Throw and Throws

class AgeValidator {
    void validateAge(int age) throws InvalidAgeException {
        if (age < 0) {
            throw new InvalidAgeException("Age cannot be negative");
        }
    }
}

class InvalidAgeException extends Exception {
    InvalidAgeException(String message) {
        super(message);
    }
}

Custom Exceptions

Java allows you to create your own exception classes by extending the Exception class.

Example: Custom Exception

class InsufficientFundsException extends Exception {
    private double amount;
    
    InsufficientFundsException(double amount) {
        this.amount = amount;
    }
    
    public double getAmount() {
        return amount;
    }
}

Exception Hierarchy

Java exceptions are organized in a hierarchy with Throwable at the top.

Exception hierarchy:

  • Throwable
    • Error
      • OutOfMemoryError
      • StackOverflowError
    • Exception
      • RuntimeException
        • NullPointerException
        • ArrayIndexOutOfBoundsException
      • IOException
      • SQLException

Exception Handling Best Practices

Best practices for exception handling:

  • Use specific exceptions instead of general ones
  • Don't catch exceptions you can't handle
  • Always clean up resources in finally block
  • Log exceptions properly
  • Don't use exceptions for flow control
  • Document all exceptions in method signatures
