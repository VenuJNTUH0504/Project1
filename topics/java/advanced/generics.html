Generics in Java

Understanding type safety and generic programming in Java

Introduction to Generics

Generics enable types (classes and interfaces) to be parameters when defining classes, interfaces, and methods.

Benefits of using generics:

  • Type safety at compile time
  • Elimination of type casting
  • Better code reusability
  • Improved code readability

Generic Classes

Generic classes are classes that can work with any data type.

Example: Generic Box Class

public class Box {
    private T content;
    
    public void setContent(T content) {
        this.content = content;
    }
    
    public T getContent() {
        return content;
    }
}

// Usage
Box stringBox = new Box<>();
stringBox.setContent("Hello");
String value = stringBox.getContent();

Box intBox = new Box<>();
intBox.setContent(42);
int number = intBox.getContent();

Generic Methods

Generic methods are methods that can work with any data type.

Example: Generic Utility Methods

public class Utils {
    public static  T getFirstElement(List list) {
        if (list.isEmpty()) {
            return null;
        }
        return list.get(0);
    }
    
    public static  void printPair(T first, U second) {
        System.out.println("First: " + first + ", Second: " + second);
    }
}

// Usage
List names = Arrays.asList("Alice", "Bob", "Charlie");
String first = Utils.getFirstElement(names);

Utils.printPair("Age", 25);

Bounded Type Parameters

Bounded type parameters restrict the types that can be used as type arguments.

Example: Bounded Type Parameter

public class NumberBox {
    private T number;
    
    public NumberBox(T number) {
        this.number = number;
    }
    
    public double getDoubleValue() {
        return number.doubleValue();
    }
}

// Usage
NumberBox intBox = new NumberBox<>(42);
NumberBox doubleBox = new NumberBox<>(3.14);

// This would cause a compile error
// NumberBox stringBox = new NumberBox<>("Hello");

Wildcards

Wildcards provide flexibility when working with generic types.

Example: Wildcard Usage

public class WildcardExample {
    // Upper bounded wildcard
    public static double sumOfList(List list) {
        double sum = 0.0;
        for (Number n : list) {
            sum += n.doubleValue();
        }
        return sum;
    }
    
    // Lower bounded wildcard
    public static void addNumbers(List list) {
        for (int i = 1; i <= 10; i++) {
            list.add(i);
        }
    }
}

// Usage
List integers = Arrays.asList(1, 2, 3);
List doubles = Arrays.asList(1.1, 2.2, 3.3);

double sum1 = WildcardExample.sumOfList(integers);
double sum2 = WildcardExample.sumOfList(doubles);

List numbers = new ArrayList<>();
WildcardExample.addNumbers(numbers);

Type Erasure

Type erasure is the process by which the Java compiler removes all information related to type parameters and type arguments.

Example: Type Erasure

// At compile time
List stringList = new ArrayList<>();
List intList = new ArrayList<>();

// After type erasure
List stringList = new ArrayList();
List intList = new ArrayList();

Important points about type erasure:

  • Generic types exist only at compile time
  • Type information is erased at runtime
  • Cannot use instanceof with parameterized types
  • Cannot create arrays of parameterized types

Generics Best Practices

Best practices for using generics:

  • Use descriptive type parameter names (T, E, K, V)
  • Prefer generic methods over raw types
  • Use bounded wildcards to increase API flexibility
  • Avoid using raw types in new code
  • Consider using generic methods for static utility methods
  • Document generic type parameters in Javadoc
