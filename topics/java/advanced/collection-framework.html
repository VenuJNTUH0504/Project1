Collection Framework in Java

Understanding Java's powerful collection framework and its implementations

Introduction to Collections

The Java Collection Framework provides a unified architecture for storing and manipulating groups of objects.

Key interfaces in the Collection Framework:

  • Collection (root interface)
  • List (ordered collection)
  • Set (unique elements)
  • Queue (FIFO structure)
  • Map (key-value pairs)

List Interface

List is an ordered collection that allows duplicate elements.

Example: ArrayList

List fruits = new ArrayList<>();
fruits.add("Apple");
fruits.add("Banana");
fruits.add("Orange");

// Access elements
String firstFruit = fruits.get(0);
fruits.set(1, "Mango");

// Iterate
for (String fruit : fruits) {
    System.out.println(fruit);
}

Common List implementations:

  • ArrayList (resizable array)
  • LinkedList (doubly-linked list)
  • Vector (thread-safe array)

Set Interface

Set is a collection that contains no duplicate elements.

Example: HashSet

Set numbers = new HashSet<>();
numbers.add(1);
numbers.add(2);
numbers.add(3);
numbers.add(1); // Duplicate, won't be added

// Check existence
boolean containsTwo = numbers.contains(2);

// Remove element
numbers.remove(3);

Common Set implementations:

  • HashSet (unordered, uses hash table)
  • TreeSet (sorted, uses red-black tree)
  • LinkedHashSet (maintains insertion order)

Map Interface

Map stores elements as key-value pairs, where each key is unique.

Example: HashMap

Map ageMap = new HashMap<>();
ageMap.put("John", 25);
ageMap.put("Alice", 30);
ageMap.put("Bob", 28);

// Access value
int johnsAge = ageMap.get("John");

// Check key existence
boolean hasAlice = ageMap.containsKey("Alice");

// Remove entry
ageMap.remove("Bob");

Common Map implementations:

  • HashMap (unordered, uses hash table)
  • TreeMap (sorted, uses red-black tree)
  • LinkedHashMap (maintains insertion order)

Queue Interface

Queue represents a collection designed for holding elements prior to processing.

Example: PriorityQueue

Queue priorityQueue = new PriorityQueue<>();
priorityQueue.offer(5);
priorityQueue.offer(1);
priorityQueue.offer(3);

// Process elements
while (!priorityQueue.isEmpty()) {
    int element = priorityQueue.poll();
    System.out.println(element);
}

Common Queue implementations:

  • PriorityQueue (priority-based queue)
  • ArrayDeque (double-ended queue)
  • LinkedList (can be used as queue)

Comparators and Sorting

Java provides ways to sort collections using Comparable and Comparator interfaces.

Example: Custom Sorting

// Using Comparable
class Person implements Comparable {
    private String name;
    private int age;
    
    @Override
    public int compareTo(Person other) {
        return this.age - other.age;
    }
}

// Using Comparator
Comparator nameComparator = (p1, p2) -> 
    p1.getName().compareTo(p2.getName());

// Sort list
List people = new ArrayList<>();
Collections.sort(people); // Natural ordering
Collections.sort(people, nameComparator); // Custom ordering

Collection Framework Best Practices

Best practices for using collections:

  • Choose the right collection type for your needs
  • Use generics to ensure type safety
  • Consider thread safety requirements
  • Use immutable collections when possible
  • Initialize collections with appropriate capacity
  • Use enhanced for loop for iteration
