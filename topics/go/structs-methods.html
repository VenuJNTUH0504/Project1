Understanding structs and methods in Go

Struct Basics

Structs are composite data types in Go:

// Basic struct
type Person struct {
    Name string
    Age  int
}

// Struct initialization
p := Person{
    Name: "John",
    Age:  30,
}

// Anonymous struct
p := struct {
    Name string
    Age  int
}{
    Name: "John",
    Age:  30,
}

Struct Embedding

Go supports embedding of structs:

// Base struct
type Animal struct {
    Name string
    Age  int
}

// Embedded struct
type Dog struct {
    Animal
    Breed string
}

// Using embedded struct
d := Dog{
    Animal: Animal{
        Name: "Rex",
        Age:  5,
    },
    Breed: "German Shepherd",
}

Methods

Methods are functions with a receiver:

// Method with value receiver
func (p Person) Greet() string {
    return fmt.Sprintf("Hello, my name is %s", p.Name)
}

// Method with pointer receiver
func (p *Person) Birthday() {
    p.Age++
}

// Using methods
p := Person{Name: "John", Age: 30}
fmt.Println(p.Greet()) // Hello, my name is John
p.Birthday()
fmt.Println(p.Age) // 31

Method Sets

Understanding method sets in Go:

// Value receiver method set
type Circle struct {
    Radius float64
}

func (c Circle) Area() float64 {
    return math.Pi * c.Radius * c.Radius
}

// Pointer receiver method set
func (c *Circle) Scale(factor float64) {
    c.Radius *= factor
}

// Using method sets
c := Circle{Radius: 5}
fmt.Println(c.Area()) // 78.53981633974483
c.Scale(2)
fmt.Println(c.Area()) // 314.1592653589793

Interfaces and Structs

Structs can implement interfaces:

// Interface definition
type Shape interface {
    Area() float64
    Perimeter() float64
}

// Struct implementing interface
type Rectangle struct {
    Width  float64
    Height float64
}

func (r Rectangle) Area() float64 {
    return r.Width * r.Height
}

func (r Rectangle) Perimeter() float64 {
    return 2 * (r.Width + r.Height)
}

// Using interface
var s Shape = Rectangle{Width: 5, Height: 3}
fmt.Println(s.Area())      // 15
fmt.Println(s.Perimeter()) // 16

Best Practices

  • Use meaningful field names in structs
  • Keep structs focused and cohesive
  • Use pointer receivers when modifying state
  • Use value receivers for immutable operations
  • Consider embedding for code reuse
  • Document exported structs and methods
  • Use interfaces for abstraction
  • Keep method sets small and focused
