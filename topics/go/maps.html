Go Maps

Understanding maps in Go

Map Basics

Maps are Go's built-in associative data type:

// Map declaration
var m map[string]int

// Map initialization
m = make(map[string]int)

// Map literal
m := map[string]int{
    "one": 1,
    "two": 2,
    "three": 3,
}

// Map operations
m["four"] = 4
delete(m, "two")
value, exists := m["three"]

Map Operations

Common operations on maps:

// Adding elements
m := make(map[string]int)
m["one"] = 1
m["two"] = 2

// Updating elements
m["one"] = 10

// Deleting elements
delete(m, "two")

// Checking existence
value, exists := m["one"]
if exists {
    fmt.Println("Value:", value)
}

// Length of map
length := len(m)

Map Iteration

Different ways to iterate over maps:

// Basic iteration
m := map[string]int{
    "one": 1,
    "two": 2,
    "three": 3,
}

// Iterate over keys and values
for key, value := range m {
    fmt.Printf("Key: %s, Value: %d\n", key, value)
}

// Iterate over keys only
for key := range m {
    fmt.Println("Key:", key)
}

// Iterate over values only
for _, value := range m {
    fmt.Println("Value:", value)
}

Map of Maps

Creating and using nested maps:

// Map of maps
m := make(map[string]map[string]int)

// Initialize inner map
m["outer"] = make(map[string]int)
m["outer"]["inner"] = 42

// Map of slices
m := make(map[string][]int)
m["numbers"] = []int{1, 2, 3}

// Map of structs
type Person struct {
    Name string
    Age  int
}
m := make(map[string]Person)
m["john"] = Person{Name: "John", Age: 30}

Map Concurrency

Handling maps in concurrent code:

// Using sync.Map for concurrent access
var m sync.Map

// Store values
m.Store("key", "value")

// Load values
value, ok := m.Load("key")

// Delete values
m.Delete("key")

// Range over sync.Map
m.Range(func(key, value interface{}) bool {
    fmt.Println(key, value)
    return true
})

Best Practices

  • Initialize maps before use
  • Check for existence when accessing map values
  • Use appropriate key types
  • Consider map capacity for large maps
  • Use sync.Map for concurrent access
  • Be careful with map iteration order
  • Document map key requirements
  • Consider using structs for complex values
