Go Error Handling

Understanding error handling in Go

Error Basics

Errors in Go are values that implement the error interface:

// Error interface
type error interface {
    Error() string
}

// Creating errors
err := errors.New("something went wrong")
fmt.Println(err.Error()) // something went wrong

// Using fmt.Errorf
err := fmt.Errorf("invalid input: %v", input)
fmt.Println(err) // invalid input: [input value]

Error Checking

Common patterns for error checking:

// Basic error checking
file, err := os.Open("filename.txt")
if err != nil {
    log.Fatal(err)
}
defer file.Close()

// Multiple error checks
if err := doSomething(); err != nil {
    return err
}
if err := doSomethingElse(); err != nil {
    return err
}

// Error wrapping
if err != nil {
    return fmt.Errorf("failed to process: %w", err)
}

Custom Errors

Creating custom error types:

// Custom error type
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation error in %s: %s", e.Field, e.Message)
}

// Using custom error
func validate(input string) error {
    if len(input) < 5 {
        return &ValidationError{
            Field:   "input",
            Message: "too short",
        }
    }
    return nil
}

Error Wrapping

Wrapping errors with context:

// Error wrapping
func processFile(filename string) error {
    file, err := os.Open(filename)
    if err != nil {
        return fmt.Errorf("failed to open file: %w", err)
    }
    defer file.Close()

    // Process file
    return nil
}

// Unwrapping errors
if err != nil {
    if wrappedErr := errors.Unwrap(err); wrappedErr != nil {
        fmt.Println("Original error:", wrappedErr)
    }
}

Error Handling Patterns

Common error handling patterns:

// Error handling with context
func process() error {
    if err := step1(); err != nil {
        return fmt.Errorf("step1 failed: %w", err)
    }
    if err := step2(); err != nil {
        return fmt.Errorf("step2 failed: %w", err)
    }
    return nil
}

// Error handling with cleanup
func processWithCleanup() error {
    resource, err := acquireResource()
    if err != nil {
        return err
    }
    defer cleanupResource(resource)

    // Use resource
    return nil
}

Best Practices

  • Always check errors
  • Use descriptive error messages
  • Wrap errors with context
  • Create custom error types when needed
  • Use defer for cleanup
  • Handle errors at the appropriate level
  • Document error conditions
  • Consider error recovery in critical sections
