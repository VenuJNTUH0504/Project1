Writing tests and benchmarks in Go

Testing Basics

Basic testing in Go:

// Basic test function
func TestAdd(t *testing.T) {
    result := Add(2, 3)
    if result != 5 {
        t.Errorf("Add(2, 3) = %d; want 5", result)
    }
}

// Table-driven tests
func TestAddTable(t *testing.T) {
    tests := []struct {
        a, b, want int
    }{
        {1, 2, 3},
        {0, 0, 0},
        {-1, 1, 0},
    }

    for _, tt := range tests {
        if got := Add(tt.a, tt.b); got != tt.want {
            t.Errorf("Add(%d, %d) = %d; want %d", tt.a, tt.b, got, tt.want)
        }
    }
}

Benchmarking

Writing and running benchmarks:

// Basic benchmark
func BenchmarkAdd(b *testing.B) {
    for i := 0; i < b.N; i++ {
        Add(2, 3)
    }
}

// Benchmark with setup
func BenchmarkComplexOperation(b *testing.B) {
    // Setup
    data := generateData()
    
    // Reset timer after setup
    b.ResetTimer()
    
    for i := 0; i < b.N; i++ {
        ComplexOperation(data)
    }
}

// Benchmark with sub-benchmarks
func BenchmarkOperation(b *testing.B) {
    b.Run("Small", func(b *testing.B) {
        for i := 0; i < b.N; i++ {
            Operation(10)
        }
    })
    
    b.Run("Large", func(b *testing.B) {
        for i := 0; i < b.N; i++ {
            Operation(1000)
        }
    })
}

Testing Packages

Using testing packages:

// Using testify/assert
import "github.com/stretchr/testify/assert"

func TestWithAssert(t *testing.T) {
    assert.Equal(t, 5, Add(2, 3))
    assert.NotNil(t, someObject)
    assert.Contains(t, "hello", "ell")
}

// Using httptest
import "net/http/httptest"

func TestHandler(t *testing.T) {
    req := httptest.NewRequest("GET", "/", nil)
    w := httptest.NewRecorder()
    
    handler(w, req)
    
    resp := w.Result()
    assert.Equal(t, http.StatusOK, resp.StatusCode)
}

Best Practices

  • Write tests for all exported functions
  • Use table-driven tests for multiple cases
  • Keep tests simple and focused
  • Use meaningful test names
  • Include benchmarks for performance-critical code
  • Use test helpers for common setup
  • Write example tests for documentation
  • Run tests with race detector enabled
