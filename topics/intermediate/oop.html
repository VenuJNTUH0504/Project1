Object-Oriented Programming in C++

Understanding the core concepts of OOP in C++

Introduction to OOP

Object-Oriented Programming is a programming paradigm that organizes code into objects that contain data and code. It provides a clear modular structure for programs, making it excellent for large, complex, and actively maintained software systems.

Note: C++ is a multi-paradigm language that fully supports object-oriented programming principles while maintaining the efficiency of C.

Classes and Objects

Classes are the fundamental building blocks of OOP in C++. A class is a user-defined data type that encapsulates data and functions that operate on that data.

class Car {
private:
    string brand;
    int year;
    
public:
    Car(string b, int y) : brand(b), year(y) {}
    
    void displayInfo() {
        cout << year << " " << brand << endl;
    }
};

Best Practice: Use meaningful names for classes and their members. Follow a consistent naming convention throughout your code.

Encapsulation

Encapsulation is the bundling of data and the methods that operate on that data within a single unit or class. It restricts direct access to some of an object's components, which is a means of preventing accidental interference and misuse of the methods and data.

class BankAccount {
private:
    double balance;  // Private data member
    
public:
    void deposit(double amount) {
        if (amount > 0) {
            balance += amount;
        }
    }
    
    double getBalance() {
        return balance;
    }
};

Best Practice: Make data members private and provide public methods to access and modify them when necessary.

Inheritance

Inheritance allows a class to inherit attributes and methods from another class. This promotes code reuse and establishes a relationship between parent and child classes.

class Animal {
public:
    virtual void makeSound() {
        cout << "Some sound" << endl;
    }
};

class Dog : public Animal {
public:
    void makeSound() override {
        cout << "Woof!" << endl;
    }
};

Warning: Use inheritance judiciously. Overuse of inheritance can lead to complex and hard-to-maintain code hierarchies.

Polymorphism

Polymorphism allows objects of different classes to be treated as objects of a common base class. It enables you to perform a single action in different ways.

class Shape {
public:
    virtual double area() = 0;  // Pure virtual function
};

class Circle : public Shape {
private:
    double radius;
    
public:
    Circle(double r) : radius(r) {}
    
    double area() override {
        return 3.14159 * radius * radius;
    }
};

Note: Virtual functions enable runtime polymorphism in C++. The override keyword helps catch errors and improves code readability.

Abstraction

Abstraction means hiding complex implementation details and showing only the necessary features of an object.

class Database {
private:
    // Complex implementation details
    void connectToServer();
    void authenticateConnection();
    void executeQuery();
    
public:
    // Simple interface for users
    bool save(const Data& data) {
        connectToServer();
        authenticateConnection();
        return executeQuery();
    }
};

Best Practice: Use abstraction to hide complexity and provide simple interfaces for users of your classes.
