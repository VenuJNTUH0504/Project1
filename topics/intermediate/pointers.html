Pointers & References in C++

Understanding memory management and indirect access in C++

Introduction to Pointers

Pointers are variables that store memory addresses of other variables. They are powerful features in C++ that enable direct memory manipulation and efficient programming.

Note: Pointers are one of the most powerful but also most dangerous features in C++. Use them carefully to avoid memory leaks and undefined behavior.

int number = 42;
int* ptr = &number;  // ptr stores the address of number
cout << *ptr;       // Prints 42 (dereferencing)

Dynamic Memory Allocation

Dynamic memory allocation allows you to allocate memory at runtime using the new operator. This memory must be manually deallocated using delete.

// Single object allocation
int* ptr = new int(42);
delete ptr;  // Don't forget to deallocate!

// Array allocation
int* arr = new int[5];
delete[] arr;  // Use delete[] for arrays

Warning: Always match new with delete and new[] with delete[] to avoid memory leaks.

References

References provide an alternative way to alias existing variables. Unlike pointers, references must be initialized when declared and cannot be null.

int number = 42;
int& ref = number;  // ref is an alias for number
ref = 24;          // number is now 24

Best Practice: Use references instead of pointers when you don't need reassignment or null values.

Smart Pointers

Smart pointers are wrapper classes that manage memory automatically. They help prevent memory leaks and make code safer.

#include 

// Unique pointer (exclusive ownership)
std::unique_ptr uptr = std::make_unique(42);

// Shared pointer (shared ownership)
std::shared_ptr sptr = std::make_shared(42);

Modern C++: Prefer smart pointers over raw pointers for automatic memory management.

Common Pitfalls

Here are some common issues to watch out for when working with pointers:

  • Dereferencing null pointers
  • Memory leaks (forgetting to delete)
  • Dangling pointers (pointing to deleted memory)
  • Double deletion

Warning: Always initialize pointers and check for null before dereferencing.

// Bad practice
int* ptr;           // Uninitialized pointer
*ptr = 42;         // Undefined behavior!

// Good practice
int* ptr = nullptr;  // Initialize to nullptr
if (ptr) {          // Check before using
    *ptr = 42;
}
