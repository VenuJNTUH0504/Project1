Standard Template Library (STL) in C++

Understanding the powerful collection of template classes and functions

Introduction to STL

The Standard Template Library (STL) is a powerful set of C++ template classes that provide general-purpose classes and functions with templates that implement many popular and commonly used algorithms and data structures.

Note: STL components are highly optimized and provide excellent performance. They are also type-safe and generic.

Containers

STL containers are data structures that store collections of objects. They are implemented as class templates, which allows them to be used with any type.

#include 
#include 
#include 

// Vector (dynamic array)
vector numbers = {1, 2, 3, 4, 5};

// List (doubly linked list)
list names = {"Alice", "Bob", "Charlie"};

// Map (key-value pairs)
map ages = {
    {"Alice", 25},
    {"Bob", 30},
    {"Charlie", 35}
};

Best Practice: Choose the appropriate container based on your specific needs for insertion, deletion, and access patterns.

Algorithms

STL provides a rich set of algorithms that can be used with containers. These algorithms are implemented as function templates.

#include 

vector numbers = {5, 2, 8, 1, 9};

// Sort the vector
sort(numbers.begin(), numbers.end());

// Find an element
auto it = find(numbers.begin(), numbers.end(), 8);

// Transform elements
transform(numbers.begin(), numbers.end(), 
          numbers.begin(), [](int x) { return x * 2; });

Note: STL algorithms work with iterators, making them extremely flexible and reusable.

Iterators

Iterators are objects that point to elements in a container. They provide a way to access and traverse container elements.

vector numbers = {1, 2, 3, 4, 5};

// Using iterators
for (auto it = numbers.begin(); it != numbers.end(); ++it) {
    cout << *it << " ";
}

// Range-based for loop (C++11)
for (int num : numbers) {
    cout << num << " ";
}

Warning: Be careful when modifying containers while iterating through them, as it can invalidate iterators.

Function Objects

Function objects (functors) are objects that can be called like functions. They are often used with STL algorithms.

// Function object
struct Square {
    int operator()(int x) const {
        return x * x;
    }
};

vector numbers = {1, 2, 3, 4, 5};
Square square;

// Using function object
transform(numbers.begin(), numbers.end(), 
          numbers.begin(), square);

// Using lambda (C++11)
transform(numbers.begin(), numbers.end(), 
          numbers.begin(), [](int x) { return x * x; });

Modern C++: Prefer lambda expressions over function objects when the logic is simple and used only once.
