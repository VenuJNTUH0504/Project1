C++ Basic Syntax

Understanding the fundamental building blocks of C++ programs

Program Structure

A C++ program consists of various components, including:

  • Preprocessor directives
  • The main function
  • Statements and expressions
  • Comments
  • Variables and data types
  • Functions
  • Classes and objects

Here's a simple C++ program that demonstrates the basic structure:

// This is a comment
#include <iostream>  // Preprocessor directive

// The main function - entry point of the program
int main() {
    // Statement that prints "Hello, World!" to the console
    std::cout << "Hello, World!" << std::endl;
    
    // Return statement
    return 0;
}

Comments

Comments are used to add explanatory notes to your code. They are ignored by the compiler.

Single-line Comments

// This is a single-line comment

Multi-line Comments

/* This is a multi-line comment.
   It can span multiple lines.
   The compiler ignores everything between the opening and closing markers. */

Best Practice: Use comments to explain why you're doing something, not what you're doing (the code itself should be clear enough to show what's happening).

Variables and Data Types

Variables are named storage locations that hold data. Before using a variable in C++, you must declare it with a specific data type.

Basic Data Types

Data Type Description Size (typical) Example
int Integer (whole number) 4 bytes int age = 25;
float Single-precision floating-point 4 bytes float price = 19.99f;
double Double-precision floating-point 8 bytes double pi = 3.14159265359;
char Single character 1 byte char grade = 'A';
bool Boolean (true/false) 1 byte bool isActive = true;

Variable Declaration and Initialization

// Declaration
int age;

// Declaration with initialization
int score = 100;

// Multiple variables of the same type
int x = 10, y = 20, z = 30;

// Constants (values that cannot be changed)
const double PI = 3.14159265359;

Warning: In C++, variables must be declared before they are used. Using an undeclared variable will result in a compilation error.

Type Modifiers

Type modifiers can be used to alter the size and range of basic data types:

Modifier Description Example
short Reduces the size of an integer short int smallNumber = 100;
long Increases the size of an integer or double long int bigNumber = 1000000L;
unsigned Allows only positive values, doubling the positive range unsigned int positiveOnly = 100;
signed Allows both positive and negative values (default) signed int withSign = -100; 
// Examples of type modifiers
short int smallNumber = 32767;
long int bigNumber = 2147483647L;
unsigned int positiveOnly = 4294967295U;
long double preciseValue = 3.14159265359L;

Derived Data Types

C++ also supports several derived data types:

Arrays

// Array declaration and initialization
int numbers[5] = {1, 2, 3, 4, 5};
char name[10] = "C++";

// Accessing array elements (zero-indexed)
int firstNumber = numbers[0];  // Gets the first element (1)

Pointers

// Pointer declaration and initialization
int number = 42;
int* ptr = &number;  // ptr holds the memory address of number

// Dereferencing a pointer (accessing the value it points to)
int value = *ptr;  // value is now 42

References

// Reference declaration and initialization
int number = 42;
int& ref = number;  // ref is an alias for number

// Modifying through the reference
ref = 100;  // number is now 100 as well

Type Conversion

C++ supports both implicit (automatic) and explicit type conversions:

Implicit Conversion

int i = 10;
double d = i;  // Implicit conversion from int to double (d = 10.0)

Explicit Conversion (Casting)

// C-style cast (not recommended in modern C++)
double d = 3.14;
int i = (int)d;  // i = 3

// C++ style casts
double value = 3.14;
int intValue = static_cast(value);  // Preferred method in modern C++

Best Practice: Use C++ style casts (static_cast, dynamic_cast, const_cast, reinterpret_cast) instead of C-style casts for better type safety and clarity.

Operators

C++ provides a rich set of operators for performing various operations:

Arithmetic Operators

int a = 10, b = 3;
int sum = a + b;        // Addition: 13
int difference = a - b; // Subtraction: 7
int product = a * b;    // Multiplication: 30
int quotient = a / b;   // Division: 3 (integer division)
int remainder = a % b;  // Modulus: 1 (remainder of a/b)

Relational Operators

int x = 5, y = 10;
bool isEqual = (x == y);      // Equal to: false
bool isNotEqual = (x != y);   // Not equal to: true
bool isGreater = (x > y);     // Greater than: false
bool isLess = (x < y);        // Less than: true
bool isGreaterEqual = (x >= y); // Greater than or equal to: false
bool isLessEqual = (x <= y);  // Less than or equal to: true

Logical Operators

bool a = true, b = false;
bool andResult = a && b;  // Logical AND: false
bool orResult = a || b;   // Logical OR: true
bool notResult = !a;      // Logical NOT: false

Assignment Operators

int x = 10;
x += 5;  // Same as: x = x + 5; (x is now 15)
x -= 3;  // Same as: x = x - 3; (x is now 12)
x *= 2;  // Same as: x = x * 2; (x is now 24)
x /= 4;  // Same as: x = x / 4; (x is now 6)
x %= 4;  // Same as: x = x % 4; (x is now 2)

Increment and Decrement Operators

int a = 5;
int b = ++a;  // Pre-increment: a is incremented to 6, then b is assigned 6
int c = 5;
int d = c++;  // Post-increment: d is assigned 5, then c is incremented to 6

Input and Output

C++ uses streams for input and output operations:

Output with cout

#include <iostream>

int main() {
    int age = 25;
    std::cout << "Hello, World!" << std::endl;
    std::cout << "I am " << age << " years old." << std::endl;
    return 0;
}

Input with cin

#include <iostream>

int main() {
    int age;
    std::cout << "Enter your age: ";
    std::cin >> age;
    std::cout << "You are " << age << " years old." << std::endl;
    return 0;
}

Using the using namespace std; Directive

#include <iostream>
using namespace std;  // Makes std:: prefix unnecessary

int main() {
    int number;
    cout << "Enter a number: ";
    cin >> number;
    cout << "You entered: " << number << endl;
    return 0;
}

Warning: While using namespace std; makes code shorter, it's generally not recommended in larger projects as it can lead to naming conflicts. It's better to use the std:: prefix explicitly.

Scope and Lifetime of Variables

The scope of a variable determines where in the code it can be accessed, while its lifetime determines when it's created and destroyed.

Local Variables

void function() {
    int localVar = 10;  // Local variable, scope limited to this function
    // localVar can be used here
}
// localVar cannot be accessed here

Global Variables

int globalVar = 100;  // Global variable, accessible throughout the program

void function() {
    // globalVar can be accessed here
    globalVar = 200;
}

int main() {
    // globalVar can be accessed here too
    return 0;
}

Block Scope

int main() {
    // Outer block
    int x = 10;
    
    {
        // Inner block
        int y = 20;  // y is only accessible within this block
        x = 30;      // x is accessible here
    }
    
    // y is not accessible here
    // x is still accessible
    
    return 0;
}

Best Practice: Limit the scope of variables as much as possible to reduce complexity and potential bugs. Prefer local variables over global ones.

Naming Conventions

While C++ doesn't enforce specific naming conventions, following consistent rules makes your code more readable:

Common Naming Conventions

  • Variables and Functions: camelCase (e.g., myVariable, calculateTotal)
  • Classes: PascalCase (e.g., MyClass, StudentRecord)
  • Constants: ALL_CAPS with underscores (e.g., MAX_SIZE, PI_VALUE)
  • Private Member Variables: Often prefixed with m_ or _ (e.g., m_count, _name)

Valid and Invalid Identifiers

Rules for naming variables, functions, classes, etc.:

  • Must begin with a letter or underscore
  • Can contain letters, digits, and underscores
  • Cannot use C++ keywords
  • Case-sensitive (count and Count are different)
// Valid identifiers
int age;
double _value;
char firstName[20];
bool isValid_1;

// Invalid identifiers
int 1stNumber;    // Cannot start with a digit
float my-value;   // Cannot contain hyphens
char class;       // Cannot use C++ keywords

Preprocessor Directives

Preprocessor directives are instructions to the preprocessor, which runs before the actual compilation. They start with a # symbol.

Common Preprocessor Directives

// Include directive - includes the content of a file
#include <iostream>  // Standard library header
#include "myheader.h"  // User-defined header

// Define directive - creates a macro
#define PI 3.14159
#define SQUARE(x) ((x) * (x))

// Conditional compilation
#ifdef DEBUG
    // Code that will only be compiled if DEBUG is defined
    std::cout << "Debug mode is on" << std::endl;
#endif

// Prevent multiple inclusions of a header file
#ifndef MY_HEADER_H
#define MY_HEADER_H
    // Header file content
#endif

Note: Modern C++ often prefers const variables and inline functions over macros defined with #define for better type safety and debugging.
